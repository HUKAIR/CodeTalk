# vibetrace 外部对标 · 第六轮(对抗式幻觉检测 / 无据主张标记:能否用现有零-LLM 能力做、是否优势)

日期:2026-06-26 · 方法:`/deep-research`(105 agents,5 角度;25 claims 验证,9 finding,7 条 3-0 强证据 + 2 条 medium 综合推断;含被否决强 claim)· 三视角 · 纯外部对标。
**裁决:不力造功能。** 经 pm 五门——「多模型对抗辩论」「完备幻觉检测」**REJECT**;零-LLM 可守的那块**本质已是现有护城河**;唯一红线内新增=**dev-only 逐字引用保真自检**(并入既有 ROADMAP line 21,本研究只收紧其诚实边界);语义召回需**可选本地 NLI**=**Later**。

> 不重复前几轮(多模型对抗审查作付费产品撞 CodeRabbit/Greptile 腹地已判)。本轮专答「对抗式识别幻觉能否成 vibetrace 优势、能否用现有功能做」。

## 一、技术现状(强证据)
- **多模型对抗辩论(MAD)不值得追** `[强 3-0]`:Smit et al.「Should we be going MAD?」(ICML 2024)——MAD「不可靠地优于 self-consistency / ensembling」,Medprompt(无辩论)整体最佳且更便宜(MMLU:ChatEval 3-agent ×3-round $31.10 vs Medprompt $19.10,1.63×,分还更低);ICLR 2025 blog 在 GPT-4o-mini+Llama3.1 九 benchmark 复现「当前 MAD 无法持续优于简单单 agent test-time 策略」;最强 pro-MAD(Du 2023)被确认「算力不对等」混淆。→ **回收前几轮结论:多模型对抗不是 vibetrace 的优势点。**(源:arXiv 2311.17371、arXiv 2305.14325、ICLR25 blog)
- **SelfCheckGPT 证明可零检索/黑箱检幻觉,但需 ~20 次 LLM 采样** `[强 3-0]`:单模型高温多采样、一致=事实/发散=幻觉(EMNLP 2023,AUC-PR 93.42)。→ 非零-LLM(20× 采样),且漏「所有样本一致错」的系统性幻觉;只能作「若必须模型」备选。(源:arXiv 2303.08896)

## 二、护城河核心问题:忠实 groundedness 本质是语义蕴含(直面诚实张力)
- **学界几乎一致用模型(NLI/LLM-judge),非字符串匹配** `[强 3-0]`:ALCE(TRUE/T5-11B NLI,人类一致 85.1%)、Trust-Score(NLI 算 groundedness)、DeepEval Faithfulness(LLM 抽 claim+判矛盾)、CLATTER(claim 分解→子 claim 蕴含分类→聚合,需 LLM)。→ **零-LLM 确定性逐字重叠无法等价复刻语义蕴含式 groundedness**(我们前几轮「字符串恒等≠Trust-Score」被外部独立证实)。(源:ACL Anthology 2023.emnlp-main.398、arXiv 2409.11242、deepeval、arXiv 2506.05243)
- **纯词法重叠会系统性失效** `[强 3-0]`:HANS——BERT 在「高词法重叠但不蕴含」子集仅 0.28(低于随机);SOTA NLI 对词序乱置不敏感。→ 连专门 NLI 都被词法骗,纯字符串重叠更甚;**零-LLM 版必须诚实承认「仅逐字、高精度低召回」**。(源:arXiv 2004.11999、arXiv 1902.01007)
- **关键转向**:vibetrace 的优势不是「判主张真假(语义真值)」,而是「把答案逐字接回真实 commit/PR/会话原话(provenance,出处可追)」——**provenance 比 entailment 更适合零-LLM 确定性实现**,这是可守缝。

## 三、可行且诚实的落点(三条出路 + 边界)
- **(a) 信号真实存在** `[强 3-0]`:即便带显式引用且用强模型,**约 50% 生成未被引文完全支撑**(ALCE ELI5,citation recall 44-51%,NLI 测)。→ unsupported-claim 是高频现象,有可观测目标;但严格度量需模型,**零-LLM 只能抓「逐字无据」子集**。(源:同上 ALCE)
- **(b) correctness ≠ faithfulness(诚实上限)** `[强 3-0]`:文档支持主张是必要非充分;忠实还需「因果依赖被引文档」,而非事后凑引用(post-rationalization 在 Command-R+ 高达 57%)。→ 零-LLM 逐字 provenance 能验「这条主张被真实材料逐字支撑/无支撑」(correctness 侧),**不能宣称证明了模型的因果忠实**。(源:arXiv 2412.18004)
- **(c) 若要语义召回 → 小型本地冻结 NLI 可行(M0 允许但「有模型」)** `[强 3-0]`:HALT-RAG 用两个冻结 off-the-shelf NLI(roberta-large-mnli ~355M + deberta-v3-large ~304M)+ 词法特征 + meta-classifier,显式避免 fine-tune 大模型,QA F1 0.9786(对话仅 0.7391,靠高精度低召回+弃答);SummaC 句级 NLI 聚合到文档级。→ **「若必须模型则本地小模型可行」**,符合 M0「本地/可选/数据不出本机」;但仍是「有模型」,非零-LLM 先例。(源:arXiv 2509.07475、arXiv 2111.09525)

## 四、竞品门 / 差异化缝(medium,弱证据)
- 调研 7+ 主流忠实性/幻觉检测(ALCE/Trust-Score/DeepEval/CLATTER/HALT-RAG)**全部 model-based**;**「零-LLM 确定性逐字 provenance」是密集赛道里少被占的角度** `[medium]`。→ 既是机会(角度空白)也是警示:空白可能因「纯确定性召回太低、学界绕过」。vibetrace 可守差异化 = 不与 LLM-judge/debate 比「语义真值检出率」,而比「每条主张能否**确定性、可复现、零成本**地逐字接回真实工程材料」。**已被占的缝(不正面进):语义蕴含 groundedness、LLM-judge faithfulness、多模型 debate。**

## 五、pm 五门裁决
- **护城河门**:钉「确定性逐字溯源真实材料(commit/PR/会话原话)」= 可守;「完备幻觉检测/语义真值」需模型 = 偏离 + 撞 LLM-judge 腹地。
- **M0 红线门**:零-LLM provenance flag 红线内;语义召回若引入须**本地/可选 NLI、惰性 import、核心仍纯 stdlib、脱敏不出网**——否则破「仅 stdlib+anthropic SDK」与隐私红线。
- **证据门(N=1)**:「无据主张普遍(50%)」「词法≠蕴含」是外部强证据;但「零-LLM provenance flag 有用户价值」**无外部直证**(空白角度=未找到先例,弱证据)→ 不据此立项,先验证。
- **竞品门**:零-LLM 逐字 provenance 是空白缝,但不正面比语义检出率。
- **北极星门**:此能力服务「防被反推编造带偏」,但「标了多少无据」是产出量,**北极星仍是回面后处理率**,不为标记数背书。

**裁决:不力造新功能。**
1. **REJECT**:多模型对抗辩论(MAD,无可靠增益、贵)+「完备/语义」幻觉检测(本质需模型、撞腹地)。
2. **已是现有护城河**:零-LLM 逐字 provenance ≈ `ask`/`blame`/grounding 已做的「引真实 SHA/原话」;无需另造大功能。
3. **唯一红线内新增 = dev-only 逐字引用保真自检**:校验 vibetrace 展示的引用(SHA/PR/会话片段)是否**逐字真实存在**于真实来源,出一个可复算「X% 引用逐字可验」数substantiate 护城河。**并入既有 ROADMAP line 21 grounding-eval**——本研究只收紧其**诚实边界**:限定「标出逐字无据(高精度低召回零成本)」,**绝不宣称「检出所有幻觉/保证语义忠实」**。该项仍属「需设计/验证」,留专门设计轮,不本期 drive-by。
4. **Later/可选**:语义召回的本地小 NLI(HALT-RAG 式)——M0 允许但「有模型」,且零-LLM-only 版用户价值弱证据,**先验证再议**。

## 六、诚实边界(必须贯穿任何对外措辞)
任何 vibetrace「幻觉/无据检测」claim **必须限定**:「标出**逐字/确定性无法溯源**到真实材料的主张(高精度、低召回、零成本)」;**禁**「检出所有幻觉」「保证语义忠实」——后者本质需语义蕴含模型,与零-LLM 护城河冲突。被对抗**否决**的强 claim(供边界):「忠实细粒度检测内在必须 LLM、无确定性替代」(0-3 否决)、「support-checking 必然模型化」(1-2)——说明小模型 NLI 与确定性 provenance 的空间真实存在,但「因果忠实性可否被检测」本身仍有争议。
