# vibetrace 产品研究 Brief(资深 PM 视角)

> 证据基础:6 镜头研究(产品定义 / 用户 JTBD / 竞品两路公开网络检索 / 差距机会 / 风险 GTM)+ 仓内代码/文档核实 + 公开竞品检索。
> **核心局限:用户证据为 N=1**(单一电商数据中台开发者深访),所有"频率/金额/付费意愿"均为个人陈述,未达人群门槛(≥3 人复现)。本 brief 严格区分"有证据"与"假设/待验证"。
>
> **编者更正(2026-06-22):** 研究子 agent 的 P0-3 称"97% 接地覆盖率查无出处"——此处不准。97% 是 `spikes/grounding_coverage.py`(本周新建的 dogfood 脚本)的**实时计算输出**,是可复现指标,不是凭空数字。正确表述应为:**别把 97% 当静态对外数字引用,除非把可复算口径写进文档**——而非"无出处"。其余核实(deepseek 默认出网、三套北极星打架)属实。

## 0. 结论先行

1. **方向对、命题真,但定位叙事讲错了入口。** 用户主动找过这类工具(花 15–20h),证明痛点真实;但他找的是"读懂这段旧代码当初为什么",不是"帮我记决策"。README 首屏却主打"记决策/理解债/认知基础设施"——**这是最高杠杆、零工程成本的 P0 修复**。
2. **两条护城河,一条快被抹平,一条是真的。** "写时面包屑+读时检索"已被 Lore(arXiv git-trailer 协议)、Repowise(get_why MCP)正面占据,**不再是差异化**。真正可守的是"**零-LLM 确定性接地对抗反推式幻觉**"——正中用户"对 AI 重解释只信 6 分"的信任上限。
3. **三件事会决定生死,且都还没有数据:** ① 目标人群主力是 Cursor 还是 Claude Code(决定整条会话价值链是否落空);② 时间胶囊"回面后实际处理率"(决定唯一独占功能是真护城河还是被无视的 TODO);③ blame/接地 vs LLM 重述的真实性对比(决定能否突破 6 分信任天花板、能否收钱)。

## 1. 一句话定位 + 电梯陈述

**现状问题(有证据):** 仓内并存三套表述——"AI 编码认知层"(README:3)/"跨时间认知基础设施"(README:8)/"偿还理解债"(README:5),且都不是用户有 pull 的那句。这是定位模糊,不是丰富。

**建议收敛(更锐的版本):**

> **一句话:** vibetrace 让你问任何一行代码"当初为什么这么写",并用真实 commit、测试、PR 原话作答——而不是让 AI 事后给你编一个理由。

> **电梯陈述:** AI 帮你高速写代码,但三个月后你看不懂自己的仓库,而 AI 事后"重新解释"为什么这么设计,只是从现在的代码反推——不一定是当初的真实原因。vibetrace 读你的 git 历史和 AI 会话,把"为什么"接地到真实 SHA、测试和 PR 讨论上;`brief`/`graph`/`blame` 完全不调 LLM,关掉大模型照样能用。这是和"套壳"的分界线。

**取舍说明:** "认知层/理解债/记决策"不是错,而是**后端叙事**——降级为产品内部哲学,别放首屏。首屏只讲用户会主动搜的那句("这段代码当初为什么这么写")。理由:把已上线能力(ask/graph)和真实需求对齐,直接降低"重蹈弃用 Continue/Pieces"的风险。

## 2. ICP 与核心 JTBD(N=1 局限)

**ICP(收窄,非泛"重度 AI 编码者"):** 在**有真实业务/资金后果的中大型代码库**里、**AI 生成密度高、自己主要做 review** 的资深开发者。典型领域:订单状态机、积分、并发锁这类"决策一错就出钱"的场景。
- 依据:受访者=电商数据中台开发,最痛锚点是积分 `Math.floor→round` 超发 30 万积分/成本超 3 万;状态机重构牵 20+ 文件、漏 1 暴雷;拥有感低是"代码 AI 生成、我只 review"的副产品。
- **取舍含义:** 低后果场景(个人小项目、一眼看懂的代码)里痛点不成立。投放与落地页应主打**高后果场景(防踩坑)**,否则痛感被稀释。

**核心 JTBD:** "回头搞清这段代码/AI 决策当初为什么这么定。" 用户明确认为 **why 比 what 更痛、更常碰**(月 3-4 次)。这正是 DeepWiki 类"代码是什么"工具答不了的层,差异化空间真实存在。

**关键诚实边界(有证据):**
- vibetrace 聚焦的是用户**自排第三痛**(代码理解),不是最痛(需求变更 > 环境问题 > 代码理解)。但用户自己说"只有代码理解能靠工具改善"——取舍合理,但 **TAM/痛感叙事不能宣称命中最痛项**。
- 付费意愿挂在"**防踩坑/提效**",用户明说"专门为记录花的钱真没有"。定价文案必须落在防事故,不能落在功能/记录。

## 3. 竞争格局与真实差异化

| 对手 | 它做什么 | 对 vibetrace 的威胁 | 判断 |
|---|---|---|---|
| **Copilot Memory / Cursor Rules / Windsurf Memories** | 前向记忆,服务 agent 下次写得更好 | 低(定位分叉:前向 vs 回溯) | 短期不正面撞。但它们已握有会话+git+diff 全上下文,扩到"回溯解释"只是**一个功能迭代的距离** |
| **Lore(arXiv 2603.15566)** | git trailer 决策记录标准,local-first、零-LLM 可查 | **高** | 与 Vibe-Decision/Vibe-Watch 几乎同一想法,有学术背书+开放标准潜力+先天 agent-friendly。**面包屑不该当独占壁垒** |
| **Repowise(get_why MCP)** | 多源挖决策、证据标 verified/fuzzy、映射代码图、MCP 分发 | **高** | 近乎正面对标。但走 AGPL 开核+云托管+团队向(PR Bot/bus factor),让出了"纯本地+单人"缝 |
| **Pieces** | on-device ML、9 个月时间记忆、自动脱敏、"不出本机除非显式开云" | **中-高(隐私维度)** | 在 local-first 上比 vibetrace **更彻底**。vibetrace 默认出网 deepseek,隐私卖点会被压制 |
| **Unblocked / Swimm** | 团队/云/多源(Slack+Jira+PR)institutional knowledge | 低 | 重型团队方案,让出单人+本地缝。但若推轻量个人版/MCP 单机,缝会变窄 |
| **Claude Code 的 git 考古 skill** | agent 直接跑 git 考古、从 diff 反推 why | 中 | 零成本替代,但只能**从 diff 反推**——正是用户只信 6 分的痛点。vibetrace 的写时面包屑是它给不出的 |

**护城河会不会被 IDE 大厂一个功能抹平?判断:**

- **"写时捕获+读时检索"组合:会,且正在被抹平。** Lore/Repowise 已用几乎相同措辞。停止把这个组合当卖点。
- **"本地优先":会被 Pieces 大面积抹平。** 不能当无条件卖点。
- **"零-LLM 确定性接地 + 对抗反推式幻觉":不会被轻易抹平——但原因是"大厂不愿做",不是"做不到"。** 大厂的商业模式是"让 AI 写更多代码",一个公开承认"AI 重解释只值 6 分信任、所以我们引真实 SHA 而非让 LLM 重述"的功能,**与它们的卖点冲突**。这是结构性的、而非临时的护城河——但前提是把叙事钉死在这里。
- **"时间胶囊(21 天预测-验证闭环)":目前无人做成产品**(Lore `lore stale`/Repowise 只做被动标记陈旧,不做主动到期对账)。这是 vibetrace 最该占的**概念高地**——但能否成立取决于"回面后处理率"(见风险)。

**真实护城河的来源(重新 framing):** 不是算法,是"**用得越久、本地数据资产越厚、迁移成本越高**"(SHA 缓存叙事+回填史)。"21 天"是机制不是壁垒,别当卖点过度承诺。

## 4. 功能↔需求差距 → 机会排序

**先做一次对账(有证据):** discovery 文档(gap-analysis/ROADMAP)写在 PR#28 快照,把 Cursor、PR 摄取、四层接地、北极星都列为"待建"。**但主线代码这些已落地**——经核实:`cli.py` 已有 `--source claude/cursor/both` 和 `--with-pr`;`ask.py` 已有 `format_evidence/test_refs/pr_refs` 四层接地+低置信警示;`self_report.py` 已有"回填率(北极星)"。**排期前别按过时差距重复立项**——剩余 gap 是"质量与验证",不是"从零建功能"。

### Now / P0(零或低工程成本,最高 ROI)
- **P0-1 入口叙事翻转(纯文案/IA):** README/落地页首屏从"记决策/认知债"翻成"`vibetrace ask` 这段代码当初为什么这么写"。把决策捕获降为后端。**最高杠杆动作。**
- **P0-2 信任叙事修正(纯文案):** 主打"零-LLM blame/接地引真实 SHA 对抗反推式编造",**不承诺"AI 帮你找回 why"**。能力已在(`ask.py:121` 低置信警示),缺的是叙事。
- **P0-3 给"97% 接地覆盖率"补可复算口径(见编者更正)。** 它是 `spikes/grounding_coverage.py` 的实时输出;把口径与运行方式写进文档后即可对外引用,否则不引裸数字。
- **P0-4 诚实标注出网边界。** `config.py:18` 默认 `provider=deepseek`、`base_url=api.deepseek.com`,富集/ask 把脱敏后会话+diff POST 到云端。README:8 的"完全不调用 LLM"仅对 brief/graph/debt 成立,需明确区分。

### Next / P1(验证后做,工程量小、复用已有数据)
- **P1-1 "读了哪些会话/脱敏命中"可审计视图(零-LLM):** 把"本地优先"从口号变成可核验信号,直接对症 ICP 最大采纳阻力。
- **P1-2 把零-LLM 确定性层包成 MCP server(blame/graph/ask --json):** `ask` 已有 `--json` 出口。**一次构建覆盖 Claude Code/Cursor/Windsurf 多客户端**,同时解冷启动+零 CAC 分发。比"再加一个子命令"高一个量级。**需先确认不引框架依赖(M0 红线 stdlib+anthropic)。**
- **P1-3 周报内容聚合(why+踩了哪些坑):** 对位用户高频痛(周报考古 40-60min/周)。复用已缓存逐日 narrative。**指标盯"考古时间↓"而非"周报生成数"。**
- **P1-4 回溯式影响面清单(graph/全局引用):** 对位"挖地雷连锁改动"。**与"改动前预测式影响分析"切开——前者可做,后者超红线、不做。**

### 该拒绝的(护城河纪律)
- **需求变更管理、环境排障:** 用户排第 1/第 2 痛,但他自己说"需求是人的问题、环境是运气问题",且超 M0 红线。**坚决不做**——但在 ROADMAP 显式标注是"非最痛项的合理取舍"。
- **改动前预测式影响分析、RAG/向量库、团队功能、Web 框架:** 守红线。**特别是团队版**——放宽红线=正面进 Repowise 腹地,vibetrace 在团队广度上结构性落后。
- **console 第 7 命令的并列定位:** 它与 tunnel/brief/graph 视图职责重叠。明确为"其余视图的聚合外壳",别当并列命令——命令面已过宽。

## 5. 最大风险与缓解

| 风险 | 严重度 | 缓解 |
|---|---|---|
| **冷启动空仓 + 鸡生蛋(有证据)** | 高 | 首日靠"读已有 git/会话"当场出价值(blame/ask/graph 零或低门槛),install-hook/agent-seed 定位为"后台飞轮"、绝不作首跑前置。首跑文案喊"不填 key、不留面包屑,blame/graph 当场就用" |
| **信任天花板(有证据 6 分)** | 高 | 对比 demo(同一问题:LLM 凭空重述 vs vibetrace 引真实 SHA/原话)+ dogfood 真实性验证实验。拿到证据前,6 分天花板压制付费转化 |
| **本地优先信任反噬(有证据)** | 高 | 默认出网 deepseek,而 ICP 视对话为"思维草稿、用完手动删";Pieces 隐私承诺更硬。缓解:P0-4 诚实标注 + P1-1 审计视图。注意张力:用户对纯本地体验零容忍(Ollama 一周即弃),本地优先应定位为"零摩擦默认+被动加分项",不是主动购买理由 |
| **捕获源占比未知(N=1 地基)** | 高 | 受访者主力 Cursor、决策对话大多在 Cursor 找不回。**若人群同此,整条会话价值链落空。** 缓解:问卷 2-N **必须**加"主力 AI 工具+占比",押注前最高优先 |
| **时间胶囊=被无视的 TODO(有证据)** | 高 | 用户十几个 TODO 处理不到 1/3,根因优先级博弈非可发现性。缓解:北极星改"回面后实际处理率",到期必须附"该风险已/将成事故"的证据才撬得动 |

## 6. GTM 与变现(local-first 开发者工具语境)

**GTM 主叙事:** "可核验的代码考古"(而非"AI 记忆")。对标话术:"Unblocked 给团队找回 institutional knowledge;vibetrace 给单人找回你和 AI 之间的 why。"

**分发(优先级):**
1. **MCP server > CLI(最高杠杆)。** 把 blame/graph/ask --json 包成 MCP,蹭 Claude Code/Cursor/Anthropic Skills 目录/Codex 插件的零-CAC 入口。当前只有 CLI+本地 web,把自己关在 Claude Code 用户里。
2. **开源免费做分发**,口碑驱动。

**变现(可行选项,均待 PMF 后):** 核心矛盾——用户**愿为防踩坑/提效付费,绝不为"记录决策"付费**。
- **在 dogfood 证明"真拦下一次理由丢失型踩坑"之前,不要启动任何付费墙**——会卖不动且伤口碑。
- 后续收费点排序:① 团队版"可审计接地+跨成员决策图"(但撞 Repowise,慎);② 托管富集/零配置 LLM(但与"纯本地"叙事冲突,需品牌隔离);③ 企业合规/DLP 报告。
- **定价主张必须挂"防事故结果"(拦截的事故/省回的考古工时),不能挂"功能/记录"。**

## 7. 北极星指标建议

**现状(有证据):三套口径打架** —— 代码用"回填率"(`cache.py`/`self_report.py`),交互设计 vision 用"回看率(7 天后仍被打开的日报占比)",prompt 第三套"填充率+接地覆盖率"。**三个北极星=没有北极星。**

> **北极星:回面后实际处理率** — 到期回面的风险胶囊中,用户实际采取行动(改代码/降级/标记已处理)的占比。
>
> 理由:它度量**价值兑现**而非仪式闭合。用户实测根因是优先级博弈,"回填率/回看率"只度量可发现性,会复制"TODO 全成摆设"的失败。

**支撑指标(护栏):**
1. **接地问答有用性** — ask 答案被用户标"有用/引用了真实证据"的占比。
2. **回填率(降级为仪式护栏)** — 预测-验证环是否闭上,但不作为价值度量。
3. **(最硬,定性里程碑)dogfood 真拦下一次"理由丢失型踩坑"** — 把变现从"记录"切到"防事故"的唯一硬证据,优先级高于一切过程指标。

**注意:定位收敛前不宜对外承诺任何单一数字。**

## 8. 给创始人的 3 条最重要建议(诚实、可执行)

1. **本周就翻转首屏叙事,这是零成本最高杠杆。** 把 README/落地页从"记决策/理解债/认知层"改成"问任何一行代码当初为什么这么写,用真实 SHA/测试/PR 作答"。同时给"97%"补可复算口径、补上 deepseek 出网边界说明。**理由:你已经把对的能力(ask/graph/四层接地)建好了,却用错的叙事在卖——这是在重蹈用户弃用 Continue/Pieces 的覆辙。**

2. **在写任何新功能前,先用三个验证实验把 N=1 变成 N≥3 + 把假设变成证据。** ① 问卷加"主力 AI 工具+占比"(决定 Cursor 投入);② blame/接地 vs LLM 重述真实性对比 demo(决定能否破 6 分信任天花板);③ dogfood 时间胶囊"回面后处理率 + 真拦一次踩坑"(决定唯一独占功能真伪)。**理由:这三个都还没数据,而它们每一个都能单独证伪整个产品——现在加功能是在没有地基上盖楼。**

3. **把护城河叙事从"写时面包屑+本地优先"换成"零-LLM 确定性接地对抗反推式幻觉",并尽快出 MCP server。** 前两条已被 Lore/Repowise/Pieces 占据或抹平;后者是大厂因商业模式冲突而"不愿做"的结构性窄缝,且正中用户"只信 6 分"的痛点。MCP 同时解冷启动和分发。**理由:你真正独占的东西不是你现在主打的东西——别守一条正在被填上的缝。**
