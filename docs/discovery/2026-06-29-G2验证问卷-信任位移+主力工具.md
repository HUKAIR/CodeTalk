# G2 验证问卷:接地真实记录 vs AI 反推(信任位移 + 主力工具)

> **验证问卷**(非发现式;后者走 `README.md` 的 gap-analysis SOP)。测两件**能单独证伪产品**的事:
> ① **主力 AI 工具 + 决策对话所在工具 + 能否找回**(命门,pm-brief 实验①,破 N=1);
> ② **「引真实记录 vs 纯 diff 反推」的信任差值,能否在 ≥3 人复现**(pm-brief 实验②)。
>
> **纠偏(最重要)**:别说成「破 6 分天花板」。「6 分」只是 **问卷1 单一受访者**(发现式合集章六 Q2)的真实打分(**N=1**),问卷2/3 查无打分。G2 要验的是**差值方向能否在 ≥3 人复现**,不是去破一个本就只有 N=1 的数。
> 配套刺激素材见 `2026-06-29-护城河对照卡-真实记录vs反推.md`。

## 冷外联超短版(降回复门槛:第一条只发一句)

> 陌生 / 弱关系**第一条只发开场句**,对方回了再按其答案择一两条追问,**别清单式逼问**。全程只问真实发生过的事,不解释产品、不问意愿。下面四段可直接复制。

**① 第一条(只发这句)**
> 冒昧打扰——想跟你取个具体经:你最近一次回头看代码、**想不起当初为什么那么写**、得回去考古,是怎么找回的?(或者干脆没找回?)

**② 回了 → 追命门(工具 + 会话还在不在)**
> 懂。那次你主力用哪个 AI 工具?当时和 AI 的那段决策对话,现在还**翻得到**吗,还是被新会话覆盖 / 清理了?

**③ 答得实 → 追信任锚 + 引向对照**
> 你有没有把代码贴回 AI 让它解释「当初为什么」?它说得准吗——有没有信了它结果走偏的?
> 我手上有个小对比(同一个 commit:AI 看 diff 反推的 why vs 作者当时写下的真实记录),想听你 2 分钟直觉判断哪个更可信,方便发你看下吗?

**④ 愿深聊 → 黄金版**
> (引到完整问卷 B 段;再愿意的邀 C 段:用他自己一个 commit,他作为作者判 AI 反推漏了 / 编了什么。)

**操作注记(对内)**:① 一次一问,别堆;② 盯「能报出具体项目 / 文件 / 时间」做真实性门槛,含糊就中性追问「最近具体哪一次?」**别替他举例**;③ 他夸功能 / 提需求**不接**,拉回「上次具体那次你做了啥」;④ 引到 B 段时**别**说哪份是我们的(盲法);⑤ 完整题、判据、记账见下文 A–D 段。

## 给受访者的话(可直接复制发出)
> 占你 6–10 分钟。**只问你真实经历过的具体事**,没有标准答案;诚实的「没遇到过 / 不信 / 没花过钱 / 早没了」对我们最有价值。看两段文字那题,凭直觉就好。

### A. 你与 AI 编码(只答真实发生过的)
- **A1.** 主力用哪个 AI 编码工具?各占多少比例?(若几乎只用一个,直接答「几乎只用 X」)**另:你和 AI 的「决策讨论」主要发生在哪个工具?**(可能和主力不是同一个)
- **A2.** 你和 AI 的那些决策讨论 / 对话,**现在还找得到吗**?(工具里翻得到 / 被新会话覆盖了 / 被清理了 / 没去找过)
- **A3.** 上一次你「**想不起当初为什么那么决定**」是什么时候、哪个项目?(例:为什么用 SQLite 不用 MySQL、为什么用 compare 不用减法)
  - 你怎么找回的?(多选真实做过的)**git blame / 翻 PR 讨论 / 读测试 / 问人 / 问 AI / 猜 / 放弃**
  - 最后找回了吗?
- **A4.**(承 A3)那次有没有「为什么**没**用另一个方案 / 为什么先放着」这种?找得回吗?**当初到底记下来过没有**?(记了但找不回 / 压根没记 / AI 也没问就照搬)
- **A5.** 你装过哪个「给代码补上下文 / 记忆 / 历史解释」的工具(Copilot / Continue / Pieces / Ollama / CodeGeeX…)?**为什么弃用、回退到什么**?
- **A6.** 你有没有把代码**贴回 AI 让它解释「当初为什么这么写」**?它说得对吗?**信了它结果走偏 / 返工**过吗?(具体哪次、代价多大——例:像「超发积分」那种坑)

### B.(快速热身)两段「当初为什么」(同一真实开源项目的同一 commit,不告诉你各自怎么来的)
> **这两份都可能是对的,凭直觉判断。** 一次大改动(发布前修复 + 健壮性 + 几个新功能),下面甲、乙是对「作者当初为什么这么改」的两份说法:

**版本甲**
> 把测试 mock 签名从 `*a` 改成 `*a, **k`,为未来兼容——被 mock 的函数后续可能传关键字参数,用 `**k` 吸收,省去反复改测试。

**版本乙**
> 脱敏收口在单一入口(任何来源落盘前都过一遍);course 崩溃只修签名、**不**套宽 try/except(那是代码 bug 非外部容错,吞了会掩盖真 bug);blame 复制解析而非耦合另一模块。

- **B1.** 有没有哪份让你觉得是**「编 / 瞎编」**的?具体哪句?
- **B2.** 哪份你更信是作者**当初真实**的理由?甲、乙**各打信任分(0–10)**。
- **B3.**(看完这句再答)**甲 = 只给 AI 看 diff 让它反推;乙 = 逐字引作者当初写下的真实记录。** 知道后:你对「**AI 从 diff 反推 why**」打几分?对「**逐字引真实记录**」打几分?

### C.(主推,这题最硬——你是作者,只有你知道真相)
- **C1.** 挑你最近一个**你清楚记得为什么这么写**的 commit,把 diff 发我们。**可只发结构性 diff / 把业务字段打码——我们只看你为什么这么改、不看业务内容。** 我们只把这份 diff 给 AI 让它反推 why,把结果发回给你。
- **C2.** 你来判:它命中几成?**漏了什么?编了什么?**

### D. 价值(只问实际发生过的,不问意愿)
- **D1.** 上一次那种考古,如果「为什么」能逐字引真实记录、不靠反推,**能不能让你少踩那次坑**?(别答「省多少时间」——答有没有避免一次具体返工 / 事故)
- **D2.** 你过去为「找回 / 记录决策」这类能力**实际花过钱**吗?花在哪、多少?(**别写愿不愿意付**;没花过也是有效答案)

## 发放 + 记账注记(对内,**不发给受访者**)

**「6 分」纠偏(先读)**:「AI 反推只信 6 分」是**问卷1 单一受访者**章六 Q2 的真实打分(N=1);**问卷2/3 查无打分**(问卷2 是行为级事件——信了 AI 编的「内存优化」、真因是流式、损失 2h;问卷3 是定性「Continue 经常编造、Claude Code 至少不瞎编」)。**G2 不预设 6,验的是差值方向能否在 ≥3 人复现。** 别把这三者凑成「天花板已成立」。

**盲法**:B 段发前**随机调换甲/乙顺序**;B1 前的「两份都可能对」中性铺垫**必留**(防把受访者推向「AI 那版一定差」测出假阳性——问卷1 校准证明他认可 AI 加速 what、只不信 AI 反推 why,别测成全面不信 AI)。本版 key:**甲 = AI 反推、乙 = 真实记录**(commit `4440324`,已配平两版长度降「更长≈更可信」混淆)。

**每位受访者记一行**:主力工具+占比、**决策讨论所在工具**(A1);A2 对话**载体状态**(翻得到/被覆盖/被清理/没找过);A3 找回方式(注意有没有「读测试」)+ 是否找回;A4 why-NOT 可找回 + **当初记没记**;A5 弃用过什么+为什么(per-tool 真替代 + why-NOT);A6 贴回 AI 是否被坑+代价;**B2 甲/乙信任分差**、B1 是否**自发**指出反推在编(自发=强信号);**B3「引真实」−「AI 反推」差值**;C 段作者判命中/漏/编;D1 是否避免一次坑;D2 实付金额+对象;**后果档(高/低)**——高后果(产线/钱/小时级,问卷1/2)vs 低后果(freelancer 分钟级,问卷3),同样「信任位移」商业含义不可等同。

**通过判据(到期对账,可复算)**:G2 通过 = **≥3 名受访者** 满足:① B3 中「引真实记录」信任分**显著高于**「AI 反推」(差值为正;绝对值另记、**不预设 6**);**且** ② 能自发(B1)或在 C 段指出反推的编造/遗漏。**D2 实付 = 0 不否决 G2**(信任位移是 G2;付费是 G3,二者脱钩,不挂北极星/付费墙)。任一不达 → 差值未复现,不据此启付费墙(对齐 ROADMAP:40 时序闸 G2)。

**证据纪律(承 vibetrace-pm + README SOP)**:
- A / D 走 **The Mom Test**——只采真实发生过的事,意见/意愿记 0(D2「别写愿不愿意」、D1 不诱导「省时间」——问卷1:没人为省时间买工具)。
- B / C 是**验证式刺激环节**(G2 本质是对比实验);Mom Test 的「零刺激」只约束 A/D,**不**否定 B/C 的合理性,但 B/C 有 demand characteristics,故 B1 必加中性铺垫。**C 段(受访者自己 commit、作者即真相)是唯一能证伪「AI 反推会编/会漏」的硬版本,权重高于 B**;招募时尽量把样本拉到 C(可配脱敏脚本 + 小额激励)。
- **单受访者 ≠ 人群**:升人群门槛 = 同结论 ≥3 人复现;绝不把 N<3 写成「用户们/普遍」。
- **per-tool 命门**:A1 必抽,且分清「主力工具」与「决策对话所在工具」(问卷1 主力 Cursor、部分决策在 Claude Code、且对话大多已没=生存级错位)。若 ≥3 人决策对话在 vibetrace 读不到的工具、或多答「被覆盖/清理」→ 整条会话价值链预警(pm-brief 实验①)。
- 文案落点挂「找回 why / 防踩坑」,**不**挂「记录/留痕/省时间」。
