# 护城河盲测 · 可复跑 + 干净样本验证 — 2026-06-27

**对位 pm-brief 风险**:「blame/接地 vs LLM 重述的真实性对比」是决定能否破「对 AI 重解释只信 6 分」
信任天花板、能否收钱的 kill-risk 之一(`2026-06-22-vibetrace-pm-brief.md` §5/§7)。README 原有手工
6-commit 盲测(N=1、明说无复跑脚本)。本轮把它**产品化为可复跑脚本**并在**干净样本**上验。

**工具**:`scripts/blind_test.py`(零新依赖;deepseek 反推侧 + difflib 泄漏标 + 真实记录并排)。
复跑:`python3 scripts/blind_test.py . 3`。

## 方法(诚实)

- 取带真实面包屑(Vibe-Decision/Vibe-Rejected)的 commit,把**脱敏后的纯 diff**(无 message/注释)
  喂 LLM 反推「当初为什么」,跟 vibetrace 零-LLM 引的真实面包屑并排。
- **默认挑「泄漏最低 N」= 干净样本**(why 字面不在 diff 里),而非最新 N——后者常是高自文档化
  commit、why 已夹带进 diff,反推赢得不算数(本轮修了这个选样缺陷)。
- **判对错由人**(语义对错需人/模型,脚本不自动打分;以下裁断是**我的判读、N=3、本仓**,你应复核)。
- 泄漏标 = 确定性词重叠启发式,**只测字面是否落在 LLM 看到的 diff、不测能否推断**;对短/改写 why
  偏向『未夹带』(即偏利好护城河,读时打折)。

## 语料分布发现(零-LLM,73 个带面包屑 commit)

- LLM 可见 diff(6000 字)视角:未夹带 27 / 部分夹带 43 / 已夹带 3。
- 全 diff 视角更严:仅约 18%「未夹带」。
- **结论**:vibetrace 自己这仓**高度自文档化**(why 常以注释/docstring 写进代码),**多数 commit 的 why
  已部分夹带进 diff** → 它是个**弱盲测样本**。护城河真正发威在 **why 不在代码里**的仓(多数真实代码库)。

## 结论 / 对位护城河

在 why 不在 diff 的**干净样本**上,纯 diff 反推 = **抓表层 what、漏真 why/why-NOT、编造似是而非的理由
(乃至幻觉数字、认错整条主题)**。这正是 README 的护城河主张——现在用**可复跑工具**在干净样本上验到了,
不再只是一次手工实验。对位 pm-brief「6 分信任天花板」kill-risk:**这是支撑「别信 AI 事后重述、要引真实
记录」的可复算证据**(诚实口径:N=3、本仓、我的判读、未自动打分)。

**诚实边界**:不外推别的仓;「干净」是按字面泄漏判(LLM 仍可能推断对,故 verdict 留给人);泄漏标是启发式。
要升人群结论需在多个**外部仓**复跑 + 多人独立判读。
