# vibetrace 产品方向修正意见(基于已复审的问卷1 · N=1 暂定)

> **依据**:`docs/discovery/gap-analysis-问卷1.md`(已过 4 视角对抗复审,判定 minor-revision:引述零编造、四项核心产品事实经 main 核验成立、核心判断「捕获源错位」站得住)。
> **效力**:全部为**单样本(N=1)暂定**,已标验证门槛;凡跨问卷复现或代码可证者可信度更高。**升为产品定论前须等问卷2-N**。

## 一、定位 / 入口 wedge(最高杠杆)
**修正**:入口叙事改用用户**有 pull** 的「理解旧代码 / 这段当初为什么这么写」(对应已上线的 `ask`/`graph`),把「决策捕获 / 留痕」做成**用着自然沉淀的后端**,不要求用户先认同「记录决策」的价值。
**依据**:他主动花 15-20h 找的是「读懂旧代码」类工具(问卷四 Q5);「专门为记录花的钱真没有」(问卷二 Q8)——决策记录是需被教育的供给侧需求,理解旧代码才是他会主动找的。
**动作**:落地页/README 首屏主打「为什么这么写」的接地问答,而非「帮你记决策」。
**验证**:问卷2-N 看「主动找过的工具类型」是否复现此 pull。

## 二、优先级修正(纠正 N=1 误配)
- **O5(摄取 PR/测试/需求文档)证据强度其实 > O1(Cursor)**:他「找回为什么」最强、跨六份问卷复现的来源就是 PR 讨论 + 测试用例 + 需求文档;Cursor 只是单人工具偏好。→ 提升 O5 权重(至少升到「待评估」而非 P3);明确 O1 列高是因「占比假设 + PR#29 已有码」,**非证据更强**。
- **N=1 不给人群级高分**:频率/商业影响在单样本下一律标「人群未验证」,只有「痛感/严重度」可高。
- **Cursor(O1)**:已在 **PR#29 实现**(非待建)→ 动作改为「**验证目标用户 Cursor 占比** + 评审/合并」,不是「P0 从零做」。

## 三、信任 / 隐私文案修正(对这类用户是成败)
**修正**:诚实标注「本地优先」的边界——**存储/缓存/产物本地 + 脱敏,但富集/ask/叙事仍把(脱敏后的)会话与 diff 发往云端 LLM(默认 deepseek),不是本地推理**。
**依据**:他最在意「读我跟 AI 的对话」(问卷三 Q4「思维草稿」);且隐私在 Deadline 下会失守(Q5/Q6)。
**动作**:
1. 不把「本地优先」当无条件卖点,定位为「**零摩擦默认 + 不牺牲体验**」的被动加分项(否则复制他弃用本地 Ollama 的结局);
2. 新增「**读了哪些会话 / 脱敏命中**」的可审计视图;
3. 文档/首屏的本地优先(README 已写)做小幅前移即可,别当从零补。

## 四、核心叙事的信任上限修正
**修正**:主打**零-LLM `blame` / 接地引真实 SHA**「对抗反推式幻觉」,不要承诺「AI 帮你找回 why」。
**依据**:他对「AI 事后重解释为什么」只信 **6 分**,且警惕 AI「根据现在代码反推一个理由,不一定是当初真实原因」(问卷六 Q2);他信「git blame + 翻 PR」的笨办法。
**验证**:`blame`/`ask` vs LLM 重解释的**真实性对比**实验。

## 五、功能边界修正
**修正**:**不与 Cursor 抢「生成/检索」**(他认可:@Codebase GMV 报表省 1 小时还在群里安利,问卷五 Q3),只补「**why / 决策接地**」这块未被满足的缝隙。

## 六、胶囊 / 回面修正
**修正**:回面解决「可发现性」,**不解决「优先级博弈」**——他十几个 TODO 处理不到 1/3,根因是「上线没出问题 → 排低 → 被新需求淹没」(问卷二 Q6)。
**动作**:到期时**附「这风险已/即将变成事故」的证据**(如积分 floor 那种)才撬得动;指标从「回面率」改为「**回面后实际处理率**」。
**注**:「自动到期回面 / 写时自动捕获」是**分析师推断的解法**,非用户明示需求 → 进验证计划,别当已确认。

## 七、范围取舍(诚实标注)
**修正**:显式承认他**自排最痛**是 需求变更 > 环境问题 >(代码理解);vibetrace 聚焦的是他排第三的痛——这是合理取舍(他自己说「只有代码理解能靠工具改善,需求是人的问题、环境是运气问题」问卷六 Q7),但**必须标注「非用户最痛项」**,需求变更 / 环境问题列「超产品范围 / 不做」。
**附**:被低估的「**挖地雷**」连锁改动(状态机重构牵 20+ 文件、漏 1 个上线暴雷、几个月 3-4 次,与需求变更强相关)——区分「预测(超 M0 红线,不做)」与「**回溯式影响清单**(graph / 全局引用,已有接地能力,可做)」,后者别整条压 Later。

## 八、验证前置(所有解法形态)
「自动捕获 / 自动回面 / 本地优先 / Cursor」一律**标为待验证假设**;解锁为产品结论的硬门槛:
1. **捕获源占比**:后续问卷加「主力 AI 工具 + 占比」(决定 Cursor 押注的地基);
2. **≥3 人复现**才把痛点升为人群结论;
3. **dogfood 拦截实验**:以「**真拦下一次理由丢失型踩坑**」(对位积分 `floor→round`、超发 30 万积分)为最硬指标。
