# 接地召回自证 · 行级(零 LLM,dogfood)— 2026-06-27

**问题**:`grounding_hitrate.py` 量的是 **commit 级覆盖上限**(有多少 commit *能* 被接地)。但用户是
**指着一行代码**问「这当初为什么这么写」。从没量过:**走真实 blame 引擎,行级上,vibetrace 的零-LLM
路径实际能否够到一个 authored why?** 这是 `vibetrace commands` 被搁置后换的方向——**先验证护城河本身、
再谈下游**(review 入口增强本就 gated 在「确定性引原话命中率」上)。

**脚本**:`scripts/grounding_recall.py`(零 LLM,纯本地,可复现;复用 `blame.collect_segments` 真实引擎,
非另写)。复跑:`python3 scripts/grounding_recall.py [项目] [抽样数] [seed]`。

## 方法

- ground truth 取本仓 `vibetrace/*.py` 源码非空行(40 文件、5127 行)。定种抽样 200 行(seed=1729,可复现)。
- 每行跑 `blame.collect_segments(file, line, line)`(= `line_log` top-12 行历史 → 每 commit 的
  narrative why/decisions ∪ Vibe-Decision 面包屑/evidence)。
- 判定:**该行接地可达 ⟺ 返回的任一段带 authored why**(why / decisions / evidence 任一;
  **Vibe-Watch(risks)是前瞻预测、非「为什么」,不计**——与 `grounding_hitrate` 口径一致)。
- 两档:**window** = 窗内任一段有 why(= blame **实际**行为);**strict** = 最新触达 commit(段尾)有 why。

## 结果(本仓,2026-06-27)

| 指标 | 值 | 含义 |
|---|---|---|
| 行级可达率(window) | **199/200 = 99.5%** | blame 能为该行够到一个 authored why |
| 行级可达率(strict) | 199/200 = 99.5% | 最新触达 commit 自带 why |
| window − strict | ≈ 0 | why 藏在更早 commit 的比例极小 |
| 对照:commit 级覆盖上限 | 95/124 = 76.6% | commit 加权(grounding_hitrate 口径) |

字段贡献:`decisions` 199、`why` 181、`evidence` 181、`risks`-only 0、完全无 1。

## 诚实解读(关键,勿越界)

1. **这是「可达率/上限」,不是「正确率」。** 「blame 够到一个 authored why」**≠**「够到的就是这一行
   **正确**的 why」。一个大 feature commit 改 100 行、带 1 条面包屑,这 100 行全算「可达」,但面包屑未必
   解释每一行。**「这条 why 是否真解释这一行」是语义判定、需模型——违零-LLM,R6 已钉死不做。** 故 99.5%
   是行级**可达天花板**,正如 76.6%/92% 是 commit 级**覆盖天花板**,都不是答对率。

2. **99% 饱和源于本仓严格 dogfood 面包屑纪律**(几乎每个源码 commit 都带 Vibe-Decision),**不外推**
   别的仓。换个不留面包屑的仓,行级可达率会断崖下跌——届时也只有 narrative(需 enrich/LLM)兜底。

3. **行级(99.5%)≫ commit 级(76.6%)的差**是真发现:无面包屑的 29 个 commit 多是 doc/杂务、
   几乎不碰源码行;源码行按 churn 几乎全落在带面包屑的大 feature commit 上。**commit 加权会低估
   「用户指着源码行问 why」时的实际可达率。**

## 结论 / 对下游的意义

- **零-LLM 检索**的「reach」在面包屑纪律下基本**触顶**;**绑死的约束不再是检索覆盖,而是
  「够到的 why 对这一行是否正确/具体」**——而那正是 R6 判定的、零-LLM 不该碰的语义缝。
  **故不该再往「更广检索」投入(边际≈0)**。
- **定位文案**可诚实讲「在记录存在处,我们零-LLM 确定性够到真实 authored why」;**不可**讲
  「总是给对/给全这一行的 why」。
- **review 入口增强**需要的「确定性引原话命中率」≈ 本数(带上「可达非正确」caveat):检索 reach 不是
  瓶颈 → 该项可启动,但其价值卖点须落在「逐字引真实记录」而非「判对错」。
- **面包屑纪律是这条护城河的实际燃料**:别的仓要拿到这个数,得先有写时面包屑(agent_seed)或 enrich。
