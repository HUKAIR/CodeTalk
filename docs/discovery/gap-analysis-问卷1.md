## 1. 执行摘要
这份深访**强力验证 vibetrace 核心命题**,也暴露**一个生存级错位**。

**验证(✓佐证)**:最贵的痛是"想不起当初为什么那么决定"(他:决策 why > 代码 what,"没记下来就真没了")——最严重一次 AI 用 `Math.floor` 是财务要求、理由没记,他回去改成 `round` → **超发 30 万积分、成本超 3 万**。手工留痕(博客/commit 规范/注释)全在 Deadline 下崩塌,根因"现在付出、以后才享受"。TODO/风险"全成摆设、没人提醒"。现有 AI 工具(Cursor @Codebase/Continue/CodeGeeX)全败在"为什么"、有幻觉,他信"git blame+翻PR+读测试"的笨办法。隐私上他**区别对待读代码 vs 读对话**(对话=思维草稿,更敏感,用完手动删),甚至试过本地 Ollama。→ 分别对位 vibetrace 的:决策捕获 / 写时自动捕获 / 胶囊回面 / 接地不幻觉 / 本地优先。

**生存级错位**:他**主力 Cursor,而 vibetrace 只捕获 Claude Code 会话**——他反复想找回的 AI 决策对话(Redlock→乐观锁、Kafka 状态机)全在 Cursor 里"找不到/被新会话覆盖/没导出"。按现状产品抓不到他绝大多数 AI 决策,核心价值几乎落空。其次:他找回"为什么"的最佳来源是 **PR 讨论+测试用例+需求文档**,vibetrace 都不摄取。

## 7. Top Opportunity PRD —— O2 零摩擦决策捕获 + 到期自动回面
- **问题**:Deadline 下无法靠纪律记"为什么决定",几周后无法追溯→踩坑(实证 ¥3万)。
- **核心主张**:决策"为什么"成为**默认产物**,无需手工;当初担心的风险**到期自动回面**,而非烂在 TODO。
- **要点**:① AI 写完复杂改动→自动从 会话/diff/面包屑 抽"关键决策+被否决备选";② 风险自动封时间胶囊,到期 brief/digest 回面;③ blame/ask 接地看历史决策。
- **成功指标**:决策回填率↑;**至少拦下一次"理由丢失型"踩坑**;周报考古时间↓。
- **不做**:不为单用户贸然建集成(待 O1);不做预测式影响;不引向量库。
- **依赖**:捕获源(见末节 Cursor 方向)、抽取质量。

## 附:发展 Cursor 方向的可行性(实证)
Cursor 数据目录存在;`User/globalStorage/state.vscdb` 与各 workspace 的 `state.vscdb` 是 SQLite;`cursorDiskKV` 表中 `composerData:<uuid>` 即一轮轮 AI composer 会话(实测 351 条 chat/composer 键)。
- **契合 M0 红线**:`sqlite3` 是**标准库**(零新依赖);Cursor 库在**本地**(数据不出本机);非官方 schema → 容错解析(与 Claude JSONL 同风险类)。
- **架构**:新增一个"Cursor 会话解析器"喂进现有 session→commit 富集管线(align/enrich 复用);不是重写。
- **风险**:`composerData` 格式非官方、随 Cursor 版本变,需防御式解析+版本守卫;Cursor composer 与 commit 的对齐比 Claude 松,需调匹配;**信任更敏感**(受访者明说 Cursor 对话比代码更私密)→ 本地优先保证必须做实、透明、可选。
