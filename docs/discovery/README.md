# 问卷处理 SOP(discovery questionnaire → 产品输入)

把一份**真实用户深访问卷**(填好的 `问卷N.md`)处理成可排期的产品输入,并随样本增加(N=1→N=2→…)收敛结论。本目录所有产物都按此流程产生。

> 配套:决策评审用 `.claude/skills/vibetrace-pm`(五道门);初次差距分析用 `user-product-gap-analysis` skill。证据纪律见下「硬规则」。

## 何时用
- 收到一份新的填好问卷(`~/Downloads/问卷N.md`)。
- 要把访谈洞察折进 `ROADMAP.md`,或为某机会升排期前做证据复核。

## 产物(每份问卷固定三件 + 一次 ROADMAP 折入)
- `docs/discovery/gap-analysis-问卷N.md` — 差距分析(已过对抗复审的版本)。
- `docs/discovery/修正意见-问卷N.md` — 据复审给产品/路线图的修正意见。
- `ROADMAP.md` 的「发现驱动的方向修正」段追加该问卷的折入(标注 N 与证据等级)。

## 工作流(6 步)

1. **初次差距分析** — 用 `user-product-gap-analysis`(或 `/vibetrace-pm`)跑 `问卷N.md`。走:反馈标准化 → 识别真实需求(说了什么 vs 真正要什么 vs JTBD)→ 对比当前产品(已实现?入口?摩擦?)→ 差距矩阵 → 优先级(P0/P1/拒绝)→ 产品动作 → Now/Next/Later。
2. **交叉引用(N≥2 必做)** — 把本问卷与**已有问卷**逐条对账:① 哪些痛点**corroborate**(同一痛 ≥2 人复现 → 证据等级升);② 哪些**contradict**(分歧 → 标人群异质);③ **关键画像字段**必须抽取并对比:**主力 AI 工具(Cursor/Claude Code/…)**、ICP 契合度、付费触发点。单份问卷禁止直接产出人群结论。
3. **对抗复审** — **开另一批 subagent 独立复审**(多视角:证据纪律/N 过度声称、scope 纪律(出红线/非本品域的痛不当机会)、护城河与交叉引用正确性、per-tool 影响)。判 ready / minor-revision / major。
4. **据此修订** — 按复审改 `gap-analysis-问卷N.md`,逐条标证据等级:`有证据(N=k)` / `假设(待验证)` / `查无`。
5. **起草修正意见** — `修正意见-问卷N.md`:给产品的具体动作 + 优先级 + 该拒绝项 + 验证胶囊(到期可对账)。
6. **折进 ROADMAP** — 追加「发现驱动的方向修正(问卷N · N=k)」,过 `vibetrace-pm` 五道门;升排期前须满足升级门槛。

## 硬规则(证据纪律,源自 vibetrace-pm)
- **不把单一用户当人群**:升人群结论门槛 = **同一痛点 ≥3 人复现**。未达只标"暂定/待验证"。
- **三档证据**逐条标注:有证据(N=k)/ 假设(待验证)/ 查无。引用数字必附可复算口径。
- **scope 纪律**:问卷里超 M0 红线或非本品域的痛(如 IDE 输出截断、硬件/固件、需求变更)→ 显式标"不做(合理取舍)",不当作产品机会。
- **per-tool**:主力工具分布是 vibetrace 的命门假设(决定会话价值链)。每份问卷必抽取;`不分版`的现行结论见 `vibetrace-pm` 与 PM brief,触发分版需 N≥3 的结构性需求差异或接地质量差异证据。
- **定位/变现**落点挂"防踩坑/防事故/省回考古工时",绝不挂"记录/留痕"。

## 升级门槛(从"暂定"到"排期")
某机会要进 Now/Next 排期,须:① 同类痛 ≥3 人复现;② 有可复算的成本证据(返工工时/事故);③ 落点在护城河内且过五道门。否则停在 ROADMAP「暂定」并给验证胶囊。
