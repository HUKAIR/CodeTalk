## 1. 执行摘要

这份深访(问卷2)把 vibetrace 核心命题**再验证了一遍**,并贡献两个新机制 + 一个关键画像反转。

**头号发现:主力工具反转。** 问卷2 主力 = **Claude Code**(日常编码主力、$20/月 Claude Pro),与问卷1 主力 = Cursor **正好相反**。N=2 实测工具分布异质(Claude Code 1 / Cursor 1)——这**出现了 1 个反例,提示不能默认单一主力**,但 N=2 不足以判定人群分布;**主力工具占比是 vibetrace 命门假设,列为待 N≥3 验证的头号开放问题**。

**护城河被一次真实被坑事件验证(行为层 N=1)。** 问卷六 Q2:他把一段生成器逻辑贴回 Claude 重解释,Claude 编「为了内存优化」,真因是「为了流式响应不断连」,他信了去优化内存方向、**损失 2 小时且方向走偏**。这是「反推式编造造成可量化损失」的具体事件(N=1 行为级),比问卷1「只信 6 分」的态度评分更硬。两者合起来:**「AI 重解释会编造」=有证据 N=2(2/2 复现);「编造造成实测损失」=有证据 N=1(仅问卷2)**——不要让「2/2 复现」暗示有两个等强被坑事件。

**真因藏在被 AI 丢弃的确定性记录里(新机制 N=1)。** 2048 阈值真因(防老相机 OOM,原话 `cap at 2048 to avoid OOM on old camera`)在 **Python 原型一行注释**里,AI 跨语言搬运(Python→Go)时弄丢了。任何 LLM 反推都会编「一般性最大尺寸」之类错因(他自己怕的就是被当 generic max dimension 乱调)。这是问卷1 没有的新机制(其 AI 代码是新生成非搬运);但须诚实:**这条真因在原型注释里 = 出 vibetrace 摄取源**,会话仅能定位「注释被 AI 丢了」,不直接给出真因。

**多项痛点 2/2 复现,证据等级升至「有证据 N=2」但仍 <3、不升人群结论**:why>what、手工留痕必崩、TODO 沦为噪声、因来由不清返工(均有量化事故)、周报考古、挖地雷连锁改动、隐患预感成真、主动找过 decision-memory 工具并弃用。

**护城河的诚实边界(漏读补回)**:他自己也经历过会话**物理丢失**——五月底 Flask 生成器那段 Claude Code 对话「被清理了」,为此花近 3h 硬读代码 + 自写测试、最终放弃改为重写(问卷四 Q3/Q7,他自报单次最贵 why 考古成本)。**当接地源被清理时,零-LLM 确定性接地同样无源可接,此时与 LLM 重解释拉不开差距**。这条边界必须随护城河叙事一并标注。

## 5. capture-source 发现(对「不分版」工作假设的影响)

> 措辞校正:仓内**无成文「不分版」doctrine**(README per-tool 行指向 SKILL 与 PM brief,而 SKILL 门3 把 Cursor vs Claude Code 列为**可证伪假设**、门4 列为竞品;无 PM brief 文件)。下文「不分版/双源并行」一律作**分析师采用的工作假设**陈述,不以「强化现行结论」把假设升格为定论。

**结论性观察:问卷2 主力 = Claude Code,与问卷1 主力 = Cursor 正好相反 → N=2 实测各 1,出现 1 个反例,提示不能默认单一主力;但 N=2 不足以判定人群分布,主力占比为待 N≥3 验证的头号开放问题。**

含义:

1. **对问卷2 此人:主力源 = 主线捕获源,无问卷1 式物理错位(有证据);但「核心价值不落空/零迁移」是假设(N=0,他从未试用 vibetrace)。** 复核他的 5 个 why-回溯案例:loader.go「重试一次」(问卷二 Q4/问卷四 Q3)、IMReadUnchanged(问卷一 Q6)= 真在 Claude Code 会话、ask/blame **可命中(2/5)**;2048(他主观最懊恼,问卷二 Q3)= 真因在 **Python 原型注释 = 出 vibetrace 源**,会话仅确认「AI 把注释弄丢了」(1/5 部分命中);CLAHE 2.0(问卷四 Q3)= 「原型里我没写为什么、AI 也没问就照搬」→ **任何源都没有这条 why**(0/5);Flask /tmp(他最贵 3h,问卷四 Q3/Q7)= 「那段对话被清理了」→ **Claude Code 会话已丢失**(0/5)。即**真正「会话可命中」仅 2 例**,1 例真因出域、2 例无源可接。命中率待 dogfood 实测后再定 O1 是否成立。

2. **他的会话之痛 = 难查(grep 子步 12min)+ 偶发物理丢失(漏读补回)**。前者正是 ask/blame 把 SHA→会话原话定向接地能解的(把盲搜变定位);后者(会话被清理)**ask/blame 无源可接,与 LLM 重解释拉不开差距** —— 这是护城河的诚实边界。

3. **是否分版**:N=2 仍 <3,且两人主力相反、只能算「1 个反例」。**不构成分版触发条件**。触发分版仍需 N≥3 的结构性需求差异**或**接地质量差异证据。

4. **接地质量差异不是远期门槛,是已存在的客观事实**:Cursor `files_written` 早期取自上下文文件(relevantFiles/attachedCodeChunks)而非真实编辑,会把错的 why 以 high 置信「漂白」进不可变 SHA(#30 专门修这个);Cursor composer↔commit 对齐**比 Claude 松**;composerData schema **非官方、随版本变**(见 `docs/superpowers/specs/2026-06-21-session-evidence-grounding-design.md`)。**这与样本量无关。** 结论应为「双源并行但**接地质量非对称、Cursor 需更强置信度护栏**」,而非「统一就行/双源无差别」。

5. **「双源并行已做实」补注**:Cursor 源虽 merged 入 main,但 **opt-in 默认关闭**(`config.py:22` `sources=["claude"]`)。对问卷2(主力 Claude Code)默认即读到主力源、影响小;但**主力 Cursor 用户(如问卷1)默认配置下读不到他的会话,需手动 opt-in** —— 别把「双源并行」读成「开箱对等覆盖」。

6. **会话价值链脆弱性双样本确认**:两源会话都「留不住」——Cursor 被新会话覆盖/物理丢失(问卷1),Claude Code 难查 + 偶发物理丢失且「会过期」(问卷2)。两人都说真正能依赖的是「靠近代码的留痕(人写的注释/commit 正文)」。这与 vibetrace 把决策蒸馏进 commit trailer / SHA 为键缓存的**落点方向一致**,但用户**未直接背书产品的自动 trailer 机制**(机制接受度待验证)。

## 6. 护城河证据(零-LLM 确定性接地 vs 反推式幻觉)

1. **【实测被坑事件 · N=1(仅问卷2)】** 问卷六 Q2:贴回 Claude 重解释,它编「为了内存优化」,真因是「为了流式响应不断连」,信了**损失 2 小时、优化方向走偏**。→ 「反推式编造导致真实损失」的具体行为事件,**直接证明零-LLM 确定性接地(blame 行级溯源、ask 引真实会话原话/SHA)对抗反推幻觉的价值**。**注:这是 N=1 行为级实证;与问卷1「只信 6 分」(态度评分)合起来只是「2/2 复现态度警惕 + 1 个实证事件」,不要暗示有两个被坑事件。**
2. **【真因藏在确定性记录里,反推必错 · N=1】** 问卷二 Q3:`cap at 2048 to avoid OOM on old camera` 这行注释被 AI 翻 Go 时丢了 → 任何 LLM 反推只会编「一般性最大尺寸」错因(他自己怕被当 generic max dimension 乱调)。**注:这条真因在 Python 原型注释 = 出 vibetrace 摄取源**,确定性溯源能定位「注释被丢」、但真因本体需翻原型(产品不摄取原型文件)。
3. **【会话原话是确定性接地源,主力源命中 · N=2 中此例可命中】** 问卷四 Q3:翻 Claude Code 日志看到 prompt 原话「最多重试一次不要无限阻塞」→ 确定性回答「这是 AI 定的还是我授意」,正是四层接地(会话原话)用法,源在主力 Claude Code 本机。
4. **【持久化媒介选择 · 对载体方向弱支持(非强力背书)】** 问卷二 Q8:「Claude 对话会过期、Notion 会懒得维护,只有靠近代码的留痕(注释+commit 正文)才能真正活下来」。→ 支持「**人写的注释/commit 正文比外部 Notion/会话更持久**」(有证据 N=2 方向),与 commit-trailer 落点方向一致;**但用户未直接背书「机器自动写入 commit trailer 的结构化面包屑 / SHA 为键缓存」这一具体机制**(他认可的是自己手写的正文)——该机制接受度待验证,不作「教科书案例」。

> **护城河诚实边界(必随叙事一并标)**:以上四条均以**接地源(会话/注释/commit)仍存在**为隐含前提。当 Claude Code 会话被清理(问卷四 Q3/Q7 已发生:Flask 那段被清理、为此花近 3h 最终放弃改为重写),零-LLM 接地**同样无源可接**,此时与 LLM 重解释拉不开差距。→ 连到验证计划:**接地源留存率 / 会话过期对 ask-blame 命中率的影响**,须实测,不可把有条件优势说成无条件优势。

## 9. 风险与未知(逐条标证据等级)

- **N=2 不是人群**:全部「2/2 复现」= 双样本复现、暂定 N=2、待 ≥3 升级;频率/商业影响/排期优先级均「人群未验证」。
- **主力工具占比 = 最大未知**:N=2 各 1 = 1 个反例,**不足以定人群分布**(完全可能某主力占多数)。O2 占比验证为命门。〔有证据:工具分布异质 N=2;假设:人群分布〕
