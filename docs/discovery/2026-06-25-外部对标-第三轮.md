## 一、变现 / 定价(本轮最关键,verified)

- **prosumer/自服务薄利档正被系统性放弃,定价上移到企业席位** `[商业模式]`:Sourcegraph 2025-07-23 砍 Cody Free/Pro → Enterprise $59/席 + 新 agentic 品 Amp;Tabnine 已无 $9/$12,现仅 $39(Code Assistant)/$59(Agentic),按年起步。→ **单人本地工具别押 prosumer 自服务层**,该层头部在主动放弃。
- **可持续的本地优先开源变现 = open-core 按席位订阅,首要付费闸门是『私有/公司仓库接入』** `[商业模式]`:GitKraken/GitLens 五档(Community→Pro→Advanced→Business→Enterprise,按席位),免费档「仅本地 + 公共远程」、Pro 起「支持私有仓库」。Tabby = Apache-2.0 核心 + `ee/` 专有双授权,闸门是 **SSO/LDAP/团队治理/分析**(非补全引擎本身)。→ 把核心 ask/blame/接地对话保持免费 OSS,**付费只闸团队治理(SSO/成员/团队分析),与「数据不出本机」红线天然兼容**。
- **Continue(BYO-model OSS 标杆)2026-06 被 Cursor 收购并 wind-down** `[商业模式]`:定价页撤、repo 转只读。→ 别把它当「独立开源变现成功案例」;BYO/纯开源助手赛道正被大厂整合,**单点本地工具必须有非席位护城河(= 零-LLM 确定性接地)**。
- **air-gapped/on-prem 已从顶档锁定下沉为入门付费档标配** `[商业模式,medium]`:Tabnine $39 档即列「fully private deployment… air-gapped」。→ **再次印证 local-first 非可独占卖点**,必须叠零-LLM 接地差异化。

## 二、本地模型能力上限(verified,有边界)

- **Qwen2.5-Coder 0.5B–32B 全谱;32B-Instruct HumanEval 92.7% ≈ GPT-4o 92.1%**,7B 即超上代 33B DeepSeek-Coder `[技术成熟度]`。→ 「可选本地 LLM」技术可行,32B 可上 24GB 显卡。
- **边界(caveat,务必守)**:仅「代码**生成**基准」逼近;**agentic/SWE-bench 仍落后**闭源前沿;且 **「RAG 让本地模型追平云端」缺乏可靠证据**(nomic-embed-code 对标云 embedding 的 3 条 claim 全被 0-3 否决、medical-RAG 缩差距 claim 也被否)。→ 本地 LLM 定位为「**接地后的本地润色/解释**」,**不**宣传成无约束 agentic 或「本地接地=云端质量」。

## 三、接地 eval 框架(verified,且与上轮 grounding-eval 纠偏咬合)

- **DeepEval 最贴合零-LLM 逐字校验** `[技术成熟度/集成]`:原生支持非-LLM 确定性自定义 metric(继承 `BaseMetric` 实现 `measure()`、设 `self.score/self.success`,不调 LLM);自带 `ExactMatchMetric` 做严格二元 1/0 字符级相等,零 token、零 LLM。**Ragas 默认 faithfulness 必须 LLM 法官**(拆声明逐条 LLM 核验)→ 会把 LLM 注入本应零-LLM 的管线,**不适配**。
- **与第二轮 grounding-eval 纠偏的咬合(关键)**:上轮对抗复跑证伪了「对 paraphrase 答案做逐句 n-gram 归因」。本轮证据指明 DeepEval ExactMatch 的正确落点 = **校验「引用本身是否逐字源于真实记录」(citations≡material≡原始记录,C-3 已保证)** —— 字符恒等在这条链上真成立;做成一个 **dev-only 可选 DeepEval BaseMetric**(逐字引用通过率基线),**不**入核心依赖(守 M0 stdlib)。开放问:ExactMatch 默认零容忍,是否需空白/标点归一化避免假阴(openQuestion)。

## 四、对 vibetrace 的可落地结论

1. **变现路径(verified)**:核心(ask/blame/graph/search/接地对话)免费 OSS;付费闸门 = **团队治理(SSO/成员/团队分析)+ 企业 on-prem**,按席位;**不做 prosumer 个人薄利档**。护城河仍是零-LLM 确定性接地(非席位、非本地)。变现在 dogfood 证「真拦一次理由丢失型踩坑」前不启动(沿用 pm 门)。
2. **本地 LLM(verified 可行)**:`ollama` provider 开关可建(已在缺口方案);定位「本地润色/解释」,文案守边界(生成≈云、agentic/检索仍逊)。
3. **接地 eval(verified 路径)**:做 dev-only DeepEval `ExactMatchMetric` 校验「逐字引用通过率」(校引用真实性,非答案归因);不接 Ragas(LLM-judge)。
4. **MCP 分层(仍盲)**:留第四轮专攻「在 GitHub MCP 之上做 provenance 增强层的先例 + 共存 vs 竞争」。
