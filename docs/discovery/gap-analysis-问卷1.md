**矩阵注记(据复审修订):**

- **胶囊回面收益下调**:问卷二 Q6 明说即便提醒 TODO 也大概率被无视——根因是**优先级博弈**(「上线后没出问题→优先级排低→新需求一个接一个」,十几个 TODO 处理不到 1/3),**不是没被提醒**。自动回面解决「可发现性」,不解决「优先级」。真正能撬动他的是「到期时附带『这个风险已经/即将变成事故』的证据(如积分 floor 那种)」。**验证指标从「回面率」改为「回面后实际处理率」**(否则复制他口中「TODO 全成摆设」的失败)。
- **O5(PR/测试/文档)对该受访者证据强度 > O1(Cursor)**:他「找回为什么」最强、跨六份问卷复现的来源就是 **PR 讨论+测试用例+需求文档**(问卷一 Q3、二 Q3、四 Q2/Q8、五 Q6/Q8 几乎每次都走这条);而「主力 Cursor」虽真实,是**单人工具偏好**。**O1 列 P0 是出于「工具源占比假设 + PR#29 已有代码」,非证据更强**——勿读成 P0=证据最强。
- **本地优先的出网边界(诚实标注)**:产品「本地优先」指**存储/缓存/产物不出本机 + 脱敏**(README 第 3、85-91 行 + `redact_secrets` 已证);但**富集/ask/叙事仍把(脱敏后的)会话与 diff 文本 POST 到云端 LLM**(`config.py` 默认 `provider=deepseek`、`base_url=api.deepseek.com`;`llm.py` 把 system+messages POST 到 `/chat/completions`)——**默认是 deepseek 云端,非本地推理**。对最在意「读我跟 AI 对话」的这位受访者(问卷三 Q4「思维草稿」),把「本地优先」当无条件信任信号而不点明会话文本出网,有误导风险,必须诚实标注。
- **本地体验零容忍(反向约束)**:他对纯本地方案体验零容忍——2024-04 折腾 3 天本地 CodeLlama+Continue+Ollama 70B,「安全性 100% 但用着太憋屈」,一周即弃(问卷三 Q3)。故「本地优先」**必须不牺牲体验**,否则同样被弃。
- **「改动后影响清单」拆分**:区分「**预测(超 M0 红线,不做)**」与「**回溯式影响面清单(graph/全局引用,已有接地能力,可做)**」。后者正是他真实用过的笨办法(全局搜枚举引用 + 跑全量测试,问卷六 Q4)的工具化,severity 上调为 3、不应整条压 Later。

- **现状(已实现,非待建)**:enrich 已自动抽 decisions/risks、risks 已自动封 21 天胶囊到期回面、self 已有回填率北极星;故本 PRD 的**增量收窄为抽取质量 + 触发针对性 + 拦截验证**。
- **要点**:① 提升 AI 写完复杂改动→自动抽"关键决策+被否决备选"的质量与触发;② 风险胶囊**到期时附带「该风险是否已/将成事故」的证据**(否则会被新需求挤掉——问卷二 Q6);③ blame/ask 接地看历史决策,对抗反推式编造。
- **成功指标**:**回面后实际处理率↑**(非回面率);**至少拦下一次"理由丢失型"踩坑**(最硬);周报考古时间↓。
- **不做**:不为单用户贸然建集成(待 O1);不做预测式影响;不引向量库。
- **依赖**:捕获源(Cursor 见 PR#29)、抽取质量、**回面证据的撬动力**。

## 8. 验证计划
1. 捕获源分布(后续问卷加"主力 AI 工具+占比",最高优先)。
2. 痛点共性(≥3 人复现"决策why>代码what""手工留痕必崩""TODO没人提醒"才升为产品结论)。
3. 本地优先采纳实验(对隐私敏感者展示本地保证 **+ 诚实说明会话仍出网到云端 LLM**,测采纳阻力;并测「零摩擦 vs 牺牲体验」对采纳的影响——他对纯本地体验零容忍)。
4. dogfood 拦截实验(跑 O2,看是否真拦下一次理由丢失型踩坑——最硬指标;指标为**回面后处理率**)。
5. 不幻觉对比(blame/ask vs @Codebase;**新增:blame/ask vs LLM 重解释 的真实性对比**,直接对症「只信 6 分/反推式编造」)。
6. **解法形态验证**:「自动捕获/到期胶囊/纯本地」三项推断解法是否被需要(痛点已证,解法待证)。
7. **采纳 wedge 验证**:用他有 pull 的「理解旧代码/这段当初为什么」(ask/graph)作入口,O2 决策捕获作后端自然沉淀,测是否优于「先教育记录决策价值」。
8. **拥有感假设拆验**:(a)周报考古时间↓;(b)掌控感/拥有感是否真能被工具提升(后者单独验证,别默认 digest 一并解决)。

## 9. 风险与未知
- N=1,全部假设;此人极善表达,有"单一响亮用户"放大风险。
- 捕获源占比未知=最大未知(Claude Code vs Cursor)。
- 采纳悖论:他"愿为提效付费、不为记录付费"——定位须为"防踩坑/提效"。
- **需求强到他会主动找解法,但现有方案没接住**:他**确曾主动找试**此类 decision-memory 工具——2024-04 专门花两三天(约 15-20 小时)搜 GitHub/Product Hunt、试 Continue/Pieces、自写脚本想导出 Cursor 对话,**专为解决「AI 代码记不住为什么这么写」**(问卷四 Q5),都因不准/不顺手/麻烦放弃。这是对立项**关键的正向信号**(需求强到会主动找解法)。**部分真相保留**:他**装机的扳机**多是「快速理解这堆烂代码」(问卷五 Q2/Q8),即「理解旧代码」是他有 pull、会主动找的方向——但**这层不能否定 Q5 的主动搜寻事实**。
- **采纳落差(由风险升级为排序结论)**:他有 pull 的需求是「理解旧代码/这段当初为什么这么写」(对应已上线 ask/graph);「记录决策」是他**不会主动找、需被教育的供给侧推动**需求(问卷二 Q8「专门为记录花的钱真没有」)。→ **入口 wedge 用 ask/graph,O2 决策捕获作「用着用着自然沉淀」的后端**,否则重蹈他弃用 Continue/Pieces 的覆辙。
- **信任上限(证伪信号)**:他对「AI 事后重解释为什么这么设计」只信 6 分、警惕「反推式编造」(问卷六 Q2)→ 这是对「让工具找回为什么」核心叙事的**直接信任上限**;既是机会(blame 零-LLM/接地对症)也须写成**待验证的信任假设**(见 §8#5、O4 重定位)。
- **受访者自排优先级(被静默覆盖,显式补回)**:问卷六 Q1「最不想遇到」排序 = **需求变更 > 环境问题 > 老代码难理解 > AI 质量不稳**。vibetrace 聚焦的是他**排第三的痛**——这是合理产品取舍,理由是他自己说「只有代码理解我觉得能靠工具改善,需求是人的问题、环境是运气问题」(问卷六 Q7);但**必须标明非用户最痛项**。对照:
  - **需求变更(他排第一)**:**超出产品范围 / 不做**(人的问题)。
  - **环境问题(他排第二)**:**超出产品范围 / 不做**(运气问题;他说遇到环境问题那次直接停了去写文档)。
- **拥有感低 = AI 协作副产品**:他明说拥有感低是「很多代码 AI 生成、我只 review」的直接副产品(问卷六 Q6)→ **自动 digest 可能加剧而非缓解疏离**。拆两个待验证假设:(a)周报考古时间↓;(b)掌控感能否被工具提升(后者单独验证,见 §8#8)。
