## 一、MCP 分层 / 集成-vs-竞争(本轮最强信号 → 裁决:集成,有先例)

- **「在 MCP 之上做中间件/增强层」是成型赛道、有现成公开先例** `[集成策略]`:**MetaMCP**(2.5k★,被 Q1 2026 调研列 Tier 1)以 Servers→Namespaces→Endpoints 三层把多个 MCP server **聚合成单一端点 + 可插拔中间件**(内置「过滤非活跃工具」压缩 LLM 工具上下文,且「拦截并改写 MCP 请求/响应」);**mcp-proxy.dev 的 `wrapWithProxy`** 提供「拦截 `server.tool()` 注册、注入前/后置钩子、对底层工具零改动」的透明装饰器。→ **vibetrace 的 provenance/why 富集可作 namespace 级中间件插入聚合器、与 GitHub MCP 共存,不必自建路由/代理,也不必正面竞争。**
- **MCP 是事实主导标准** `[集成策略]`:UK AISI 基于 17.7 万工具背书「MCP is the dominant open standard」(97M installs、Linux Foundation 接管)→ vibetrace 押注暴露 MCP server 作集成面正确,不必自建协议。
- **新增直接竞品 GitMCP(免费开源远程 MCP)** `[集成策略]`:打「Put an end to code hallucinations」,但机制是 **retrieval-into-context(取 docs/code 进上下文、由 LLM 生成答案)**,工具仅 fetch/search_documentation,**无任何 commit/PR/session provenance 工具**,且远程/云托管。→ **恰好把 vibetrace「零-LLM 逐字 provenance 接地 + 本地」的差异化空间留出**:文案切割「我们接地的是 provenance(why)、逐字、零-LLM、本地」vs GitMCP「接地 docs/code、LLM 生成、远程」。
- ⚠ **未确证**(claims 被否):多 server 的**精确路由/命名冲突机制**(主流客户端如何在多 server 间选工具)证据不足;provenance 赛道**共存 vs 赢家通吃**悬而未决(两向强 claim 均被驳)。能确证的只是「聚合/中间件层先例存在」。

## 二、本地模型带检索接地(修正第三轮「32B≈云」)

- **接地条件下,本地与云的差距反而被拉大** `[技术成熟度]`:SWE-QA(仓库级代码问答,ACL 2026)同样 function-chunking RAG 下:Claude 3.7 Sonnet **45.28(+7.10)** vs 本地可跑 Qwen2.5-Coder-32B **38.84(+2.06)**、Qwen2.5-72B 39.34、Devstral 39.98 —— 残留 **5-6 分**,接地不能抹平。**修正第三轮「32B≈云」**:那是「无约束生成」,在「忠于来源的代码问答/解释」上本地仍逊。
- **agentic RAG 只让强云基座吃红利** `[技术成熟度]`:轻量本地模型几乎零增益甚至倒退(Qwen-72B 39.34→38.70);重 agentic 编排是云拉开差距处,不是本地小模型的机会。
- **但接地本身是凌驾一切的主导杠杆** `[技术成熟度]`:对所有模型相对无上下文 direct 均实质提升(DeepSeek V3 +8.32 / +24% 相对)。
- → **对 vibetrace**:本地 LLM(ollama)定位「**接地后的本地润色/解释、便利非对等**」,把「本地保真度仍逊于云」作前提;**强化「零-LLM 逐字接地不依赖模型质量」这条护城河逻辑**(接地是确定性、模型无关的;弱模型也不影响逐字 provenance 的正确性)。⚠ SWE-QA 测的是「接地后开放 QA」非「润色既有逐字材料」,vibetrace 最贴身的子任务仍无第三方直测(开放问)。

## 四、研究弧收口结论(四轮)
1. **MCP 路线 = 集成**(做聚合器可插的 provenance 中间件,与 GitHub MCP 共存),不竞争、不自建协议、不拼代码图谱;差异化钉「零-LLM 逐字 provenance + 本地」(GitMCP/GitKraken 占的是 retrieval/slogan,不是这格)。
2. **本地 LLM = 便利非对等**:接地是模型无关的确定性护城河;本地模型只润色,保真度逊云是前提,文案守边界。
3. **变现**:open-core 按席位 + 团队治理/on-prem 闸门、核心免费 OSS、不做个人薄利档(R3);单人工具具体营收基准 = 一手调研,非 deep-research。
4. **明确停止**:deep-research 已四轮,边际递减;剩余开放问(多 server 路由精确机制、共存 vs 赢家通吃、润色子任务实测、营收基准)需**权威 spec / 一手数据**,不再加轮。

## 待一手 / 仍开放
