## 按 G2 假设盘点(问卷123 为主)

**① 信任天花板(信不信 AI 反推 why)**
- 问卷1:章六 Q2 对「AI 重解释 why」**信任打 6 分**——`有证据(N=1)`,**这是「6 分」的唯一来源,不是人群锚**。

- 问卷3:「Continue 经常编造解释」「Claude Code 至少不会瞎编」——`有证据(N=1,定性二元)`。
- 三人**方向一致**(不信 AI 反推 why),但**只有问卷1有分数、问卷2/3 查无打分** → G2 验的是**差值方向能否 ≥3 人复现**,非「破 6 分」。问卷1 T11/T12 同时校准:**认可 AI 加速 what、只不信反推 why**(别测成全面不信 AI)。

**③ AI 编造理由**

**① 信任天花板 —— 二手最硬(人群级量化)**
- **SO 2025**(N=33,244,67.8% 回收率):highly-trust 仅 **3.1%**;distrust **45.7%** > trust **32.7%**;10+ 年经验者 highly-trust **2.5%**、highly-distrust **20.7%**(资深更不信)。`[survey 强]` survey.stackoverflow.co/2025/ai
- **SO 信任趋势(2023→2025)** :正面态度 70%+(2023/2024)→**60%**(2025);信任 ~43%(2024)→**29%**(2025,跌 11pp);采用率反升到 **84%**——**信任↓采用↑的悖论**。`[survey 强]` stackoverflow.blog/2026/02/18/closing-the-developer-ai-trust-gap/
- **SO 2025 最大痛点**:66% 头号挫败=「AI solutions **almost right but not quite**」;**75.3%** 不信 AI 答案时**转去问真人**。`[survey 强]` leaddev.com/technical-direction/trust-in-ai-coding-tools-is-plummeting
- **SO 2024**:遇困难头号问题逐字「Don't trust the output」66.2%。`[survey 强]`
- **DORA 2024**:39% 对 GenAI 输出质量只「a little / not at all」信任(页未披露 N)。`[survey 中强]` dora.dev/research/ai/trust-in-ai
- **Sonar State of Code 2025**(N>1,100):96%「不完全信任」AI 代码、53%「appears correct but unreliable」(媒体二手转述)。`[survey 强/转述中]`
→ 据此把 N=1 的「6 分」从主证据降为「我的深访也复现了人群结论」的佐证。人群信任只有 ~30%、且还在逐年下滑——**「6 分天花板」甚至偏乐观**。

**② why-loss / 决策上下文痛**

**自建补救(行为 > 抱怨,2 个工具):**
- **claude-vault**(Rust CLI → SQLite + FTS):因 Claude Code 会话 JSONL「**just gone**」,作者原话「I went to find the JSONL file. **Gone.**」;首次导入 **94,562 条消息 / 203 文件**。`[blog 一手 中]` dev.to/kuroko1t
- **Entire CLI**(git push hook → 隐藏分支存完整 AI session):「The **reasoning**, the agent's **decisions**, the **tool calls** — all **gone**」after standard commit;6 个月后调试时找不回原始 intent。`[blog 一手 中]` dev.to/stevengonsalvez

- X 一手观点(通俗背书,KOL 热度≠发生率):「长对话越聊越笨,老消息被挤出 context window 就忘了」(NetworkChuck 类)。`[X 一手观点 弱-中,URL/原话 web 初采待核]`
- **公开真空不变**:无任何带 N 的调查专测「AI 编码对话历史丢失」→ 能说「跨 5 产品被 8+ issue 反复报告」,**不能说「X% 的人丢过」**。但**密度本身是信号**:5 个主流工具全中、有人自建工具补救 = 痛点不是孤例。

**⑤ 接地 vs 反推「差值」(vibetrace 命门)—— 公开几乎为零(社交检索后仍只补到「半边」)**
- 间接旁证:SO 2025 75.3%「不信 AI 答案时会转去问真人」(意向、且是「转向人」非「转向 git 原话」,降档);arxiv 2603.17150《Intent Formalization: A Grand Challenge》把「intent 无法可靠从代码反推」立为公开难题(仅 abstract)。`[中]`
- X 一手观点(只证半边):Simon Willison 转引并赞同——「不懂代码只能反复求 AI 修 = 用另一张信用卡还债」。`[X 一手观点 中,URL web 初采待核]` ——**只证「AI 解释/修复不可信」这半边,没说「所以我去翻 git 原话」(差值的另一半仍缺)**。
- **独立工具造者(行为级 · 有人真动手造了)**:
  - **Entire CLI**:把 AI session 完整归档进 git 隐藏分支(prompts / responses / tool calls / files touched),因「standard commit 之后 reasoning、decisions、tool calls — **all gone**」,6 个月后调试无法理解 intent。`[blog 一手 中]` dev.to/stevengonsalvez
  - **git bla(i)me**:把 agent conversation traces 存为 git notes 链到行级代码,因「When you look at a line in Git blame, you usually want the **reason** behind it, not a person」;traces 成为 **stable, auditable evidence**「linked to commits with stable identifiers」。`[blog 一手 中]` ronaldtebrake.nl/blog/git-blame-intent-aware-agents/
  - → 两个工具**独立把同一痛定义为问题并自建解决方案** = 行为级验证(比抱怨强);但**都没做到「零-LLM 确定性接地」**:Entire 存 session 文本但不按 SHA 索引不去 LLM 编、git bla(i)me 存 traces 但仍需 agent 后处理。**vibetrace 的缝仍在**。
- 其它 framing 旁证:第三方工具(`git-why`)独立把问题定义为「`git blame` 给你 who/when、给不了 why」——与 ⑤ 命题逐字同构,但是工具作者营销页、非开发者发言。
- **真相更新**:仍无公开数据**直接量化**「翻 git/PR 原话 vs LLM 重述 why」的可信度差值,也几乎无人逐字说「所以我去翻 git」——但**有人造了两个工具去解决「reasoning gone」**,这比零强。**量化差值仍只能自己造数据**(对照卡 + C 段自有-commit + ≥3 真人验)。

## 结论(诚实 · 2026-06-29 第二轮全网检索后更新)
- **二手能坐实**:① 信任天花板(SO N=33k,29% 信任/46% 不信,逐年下滑)、② why-loss 痛(Atlassian N=3.5k + 学术行为观测)、③ AI 编造(CodeRabbit 1.7× + Clutch 59% + 学术)——**省掉「证明问题存在」的活,直接引用**(N=1「6 分」降为佐证;人群只有 ~30% 信任,6 分甚至偏乐观)。
- **④ 从「跨工具反复但模糊」升级为「跨 5 产品 8+ issue + 2 自建工具」**:Cursor / Claude Code CLI / Claude Code VSCode / Copilot / Copilot+Claude 全中;有人造 claude-vault(94k 条消息归档)和 Entire CLI(session 写 git 隐藏分支)补救。**仍无 N**,但密度和自建行为本身是强信号——这不是个别人的抱怨,是整个工具层的结构缺陷。对 G2 发 SOP「④ per-tool 命门」一题,可以直接**把 issue 列表当破冰卡给受访者看**:「你的主力工具上过这些 bug 吗?你的对话还在吗?」→ 比从零问高效得多。
- **⑤ 从「零」升为「有人造了工具但没做到零-LLM」**:Entire CLI / git bla(i)me 独立把「reasoning gone after commit」定义为问题并自建解决方案 = 行为级验证痛点存在;但**都不做确定性接地**(Entire 存文本不索引 SHA、git bla(i)me 存 traces 仍需 agent 后处理)——**vibetrace 的缝仍在,且更清晰了:不是「会话归档」,是「归档后零-LLM 确定性索引到行级代码」**。量化差值仍只能自造数据。
- **🚩 红线不变**:别把 survey「普遍不信 AI」当「我的差值已验证」;别把 issue 密度当发生率(N 仍=0)。
- **省力路线(更具体)**:二手坐实 ①②③(引用,零访谈)+ ④ 用 issue 列表当受访者破冰卡(省解释成本)+ ⑤ 用对照卡 + C 段自有-commit 验 → 真人只需 **1–3 人、每人只答两个问题**(④你的对话还在吗 + ⑤看完对照卡信任动几分),而非从零证明痛点。
