## 6. capture-source 发现(Claude Code 2 : Cursor 1)

**主力工具 N=3 分布 = Claude Code 2(问卷2、问卷3)/ Cursor 1(问卷1)。**

1. **对「不分版」工作假设**:N=3 仍不构成人群分布——三人主力为 CC/CC/Cursor,虽 CC 暂时领先 2:1,但 N=3 远不足以判定占比,且问卷3 的 freelancer 画像与问卷1/2 生产工程师不同,样本不可简单合并计数。触发分版仍需 N≥3 的结构性需求差异**或**接地质量差异,目前两者都未达 → **维持双源并行(工作假设),不分版**。

2. **对默认 `sources=["claude"]`(config.py:22)**:这是对 N=3 的利好——三人里两人(问卷2、问卷3)主力即 Claude Code = 默认即命中主力源、零额外配置;只有问卷1(主力 Cursor)在默认配置下读不到自己会话、需手动 opt-in。即默认配置对当前 2/3 样本友好、对 1/3 样本(Cursor 主力)有采纳摩擦。

3. **对「双源并行不分版」的最强新证据 = 问卷3 接地命中前置条件**:问卷3 的 why-丢失全发生在 Claude Code 且会话未丢失、真因就在对话里(3min 找回 Double.compare)——是三人中 ask/blame 接地**前置条件**最好的一个(问卷1 Cursor 物理丢失=错位;问卷2 部分真因在原型注释/部分会话被清理=出域)。这把「主力源对位 → 接地可命中」从问卷2 的「2/5 命中」改善为问卷3 的「前置条件高」,强化「只要捕获到主力源会话、零-LLM 接地就能命中」的核心假设——**但命中率本身仍须 dogfood 实测,且这只覆盖 CC 主力用户**。

4. **接地命中分档(dogfood 锚点,勿读成已验证强项)**:Double.compare(问卷六 Q2,673)= 会话明确可命中(用户已实测 3min 找回);SQLite 选型(问卷一 Q6,123)= 真因为客户口头约束/自身回忆,会话是否含此 why **待验证**;GridBagLayout(问卷四 Q3,456)= 用户明说「AI 没告诉我为什么选…代码里完全看不出来」,**可能会话无源**。三例命中率 dogfood 实测,不写成「高命中」整体结论。

5. **开放问题**:Cursor 主力用户(问卷1)的默认关闭摩擦 + Cursor 接地质量非对称(漂白/对齐松/schema 非官方)仍是命门,不因 CC 暂时 2:1 而消解。

## 7. ICP 异质判断(分层)

**ICP 异质,证据级=有证据 N=1(该 freelancer 变体)。** 判定:**不收窄、不盲目拓宽,建议【分层】**。

- **(a)命题层高度同质**:why>what、AI 决策失忆、手工留痕必崩、TODO 噪声、找过工具并弃用、周报考古 全部 3/3 复现,证明核心痛跨 ICP 普适,**不应收窄到只服务高后果生产**。

- **(b)价值锚/变现/定位文案须分层**:问卷1/2 的购买理由可挂「防¥3万级事故/防OOM/防产线停机」(高后果);而问卷3 后果是「客户反馈/分钟级排查」,挂事故话术对他无力。**他的真实付费触发 = 查无**——对 decision-memory/考古品类零付费历史(问卷四 Q7「没花过专门的钱」、问卷二 Q8 Continue「没花过钱」、问卷三 Q3「目前没专门为这个付过钱」),只为主力 Claude 付费(180元/月+$20/月),与问卷1/2 同构。隐私/多项目切换/跨项目汇报是他**在意的痛(有证据 N=1)**,**不等于会为其付费(查无)**。潜在付费理由若成立也只可能挂「客户代码隐私(接单生存线)」而非防事故。

- **(c)freelancer 带来两个问卷1/2 未突出的特异需求**:跨多客户项目切换捡上下文(高频)、跨项目区分决策理由(汇报时易混)——这是拓宽方向的候选,但仅 N=1,记为待验证。
