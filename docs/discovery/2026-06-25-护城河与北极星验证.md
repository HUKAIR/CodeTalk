## 二、全局叙事核验(本次新做,零 LLM)

**做什么**:`scripts/narrative_audit.py` 对缓存里**全部 68 条 commit 叙事**跑确定性核验,抓 fact-discipline 明禁的最可检子集——
叙事提到的文件 basename 若在**全仓 git 历史**(任何 commit 改过的文件)里根本不存在 → ghost(疑似编造文件名);
另以子串筛降级叙事(材料不足/富集失败)。**recall-biased screen**:宁可多报、由人裁决,与 code-review finder 同范式。

**原始筛结果**:文件名 ghost 8 条、降级 6 条。**逐条裁决后,两类全是筛子的假阳性,真违规 = 0**:

文件名 8 flag → 4 类全为良性:

| flag | token | 类别 | 裁决 |
|---|---|---|---|
| 5a7b590 / 9b75f8c | `cache.db` | 真实文件,在 `~/.vibetrace/`(不纳 git) | 良性 |
| 3712b72 / 59205b7 | `config.json` | 真实文件,在 `~/.vibetrace/` | 良性 |
| f6145eb | `postcard.html` | 代码**真实产出**的输出件(周报明信片) | 良性 |
| 607456e | `min.css`/`min.js` | glob 模式 `*.min.js`/`*.min.css`(TRIVIAL_GLOBS) | 良性 |
| a695109 | `Three.js` | 散文里的**库名**(且决策是"用零依赖**替代**它") | 良性 |
| b73c6d8 | `X/foo.py`,`tests/test_foo.py` | 源→测试映射的**占位元变量** | 良性 |

降级 6 flag → **0 条带真 `degraded` 字段**;6 条都是 decisions 4–7 条的**富叙事**,只因正文**讨论**"材料不足"这一主题被子串误捞
(如 932d328「根治…『材料不足』…的问题」、6c8d25c 自我试点发现)。→ 真降级 **0/68**。

**结论**:68 条叙事**无一编造源文件名、无一降级桩**。8+6 个 flag 全是 recall-biased 筛子按设计多报、人工裁决清掉——正是预期工作流。
**副产物(已修)**:筛子正则 `\w` 会匹配 CJK,把中文粘到尾随扩展名(「Canvas 2D替代Three.js」错切成一个 token);
真实路径不含 CJK,故路径体限 ASCII 字符类 → `2D替代Three.js` 修正为干净的 `Three.js`(单测 `test_cjk_not_glued_to_extension` 锁定)。

**边界**:本审计只查「文件名保真」这一确定性子集;**语义保真**(why 是否反推编造)不在脚本范畴,由下节盲测另证。

## 三、护城河盲测实验(diff-only 重建 vs 真实 Vibe-Decision)

**做什么**:取 6 个带真实 `Vibe-Decision` 面包屑的 commit,让一个**只看 diff**的盲测 agent 重建"当初为什么这么写",
再与作者留下的真实决策逐条比对。这是直接打"零-LLM 逐字接地 vs AI 反推式编造"这条护城河缝的自证实验。

**结果**:
- **6/6** 重建都至少有一处**编造或遗漏**。
- **2/6 完全弄错真实理由**:`ed14b3c` 漏掉真决策"刻意跳过 sha/pullNumber 字段 + 永不重命名工具名";
  `b8ced37` 把"先 vanilla-JS、React 仅在真需要时再上"的**因果讲反**。
- diff-only 还会**凭空编造被否决的备选方案**(diff 里没有的"考虑过 X 但放弃")。

**诚实 caveat**:4 个"基本命中"里有 2 个,只是因为 diff 本身**夹带了作者写的 pilot-report 文本**——不是 diff 反推出了 why,
是 why 恰好被一起 commit 进 diff 了。扣掉这层,纯 diff 反推的真实命中更低。

**最强护城河证据**:有 3 类信息 diff-only **结构性无法恢复**,正是接地不可替代之处:
1. **why-NOT / defer**:"为什么**没**这么做""为什么先缓"——diff 只录已发生的改动;
2. **diff 不可见的外部约束**:如"沙箱无 node 故走零-build vanilla-JS"——约束不在代码里;
3. **Vibe-Watch 待验证项**:作者标记的"这条到期要回看"——diff 无此维度。

→ 护城河文案应钉这三类 + "diff-only 会编造被否决备选",而非泛泛"我们记录决策"。

## 四、北极星:回面后实际处理率 = 0/0(结构性未成熟)

**定义**(依 pm-brief):到期回面的风险胶囊中,用户实际采取行动(改代码/降级/标记已处理)的占比——度量**价值兑现**,非仪式闭合。

**本仓实测 = 0/0**:不是 0%,是**分母为 0**——风险胶囊有 21 天成熟窗口,本仓最早的胶囊尚未到期回面,**还没有任何到期样本可算**。
这是该指标在新仓上的**结构性未成熟**,非产品失败。

**何时可测**:最早一批胶囊封存满 21 天后(本仓约 2026-07 上旬起陆续到期),`vibetrace watch` 会列出到期项,届时才有非零分母。
**在此之前**,北极星不可引用;护栏指标(接地命中率自证见 `-接地命中率自证.md`、dogfood 是否真拦下一次"理由丢失型踩坑")先顶上。
