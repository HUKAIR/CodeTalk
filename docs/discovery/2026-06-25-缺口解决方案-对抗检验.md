## 第三轮原始信号(⚠ 未经对抗核验,月度上限掐断 verify,仅作方向参考)

## 对抗复跑结论(2026-06-25,claude.ai 提额后真·3-lens 复跑 w2ovdxgit,33 agents,代码级核实)

月度上限提额后重跑了被掐断的 verify+synth。因 Top-3 此时已落地(#52),skeptic 实际是**审已 shipped 代码**——结果:

- **✅ 验证我的 Top-3 实现选择是对的**(且比原提案更对):review.py 用 `+++ b/`(非提案的 `diff --git`,正确处理 /dev/null/rename)、MCP **没**加第 5 工具(agent 喂 path/startLine/endLine 给既有 vibetrace_blame)、**没**改名 get_diff(GitHub MCP 无此名)。提案若被照做反而引入回退。
- **❌ grounding-eval 的「字符串 n-gram 逐句归因 + τ」主轴被三视角用本仓 fixture 实测证伪**:① 答案是 paraphrase(CHAT_SYSTEM_PROMPT 命令去 AI 腔综合)——evidence「为了流式响应不断连」→ 答案「因为要支持流式响应」,3-gram 重叠 0.286 / LCS 0.444 均 < τ0.6 → **忠实接地句被误判 unsupported(假阴)**;② **「字符串恒等满足 Trust-Score」是 overclaim**——Trust-Score 判据是语义**蕴含(entailment)**,字符串重叠抓不住改写、更抓不住语义否定(「没用装饰器」vs「用了装饰器」重叠高却相反);③ hallucination_rate 定义(unsupported 句含不存在 SHA/文件名/数字)**对旗舰幻觉失明**(问卷2「内存优化」反推编造无新 SHA/数字 → 记 0)。
  **→ 改 scope(别建被证伪的版本)**:只做「答案里**逐字命中材料的片段**高亮 + 链回其确定来源」(字符串恒等只在逐字子串处真成立)+ 保留现粗徽标 + 点开核验原话;**不打 grounded/inferred/unsupported 三标签、不设 τ**(守 ROADMAP 软对齐红线,避免把「没抄原话」误判「没依据」)。
- **定位纠偏(影响 README 文案)**:护城河文案可讲「引用是可核验的真实逐字记录(citations≡material≡原始记录,C-3 已保证)」,**不可**宣称「答案零幻觉 / 满足 Trust-Score」——答案是 paraphrase,不在逐字恒等链里。
- **横切失真**:4 个提案把已 shipped(#51/#52)当 net-new 重述——系统性失真,改判为「对既有代码的增量 delta」。ROADMAP line 15 的 sha/pullNumber 别名与代码漂移(实读仅 path/startLine/endLine)已修。
- **synth Top3 行动**:① positioning README 收窄(零代码、R2 P0,但去掉 Trust-Score overclaim);② mcp-integrate 文档收口(钉定论 + 修 line 15 漂移);③ **接地命中率 dogfood 自证 = review-entry 扩面 / grounding-eval 是否建 / local-llm / ide-ext 的共同闸门**(零-LLM、可证伪)。lore-schema(agent_seed 已要求否决备选、27/43 已内联)、ide-ext(MCP 已覆盖 IDE-context)继续 defer。
