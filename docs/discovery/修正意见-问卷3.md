## 3. capture-source 定调

- **N=3 = Claude Code 2(问卷2、问卷3)/ Cursor 1(问卷1)→ 维持双源并行、不分版**。N=3 不构成人群分布,且 freelancer 画像异质、样本不可简单合并计数;触发分版仍需 N≥3 的结构性需求差异或接地质量差异,两者均未达。
- **默认 `sources=["claude"]`(config.py:22)对当前 2/3 样本是利好**:问卷2、问卷3 主力即 Claude Code,默认零配置命中主力源;仅问卷1(Cursor 主力)默认读不到、需 opt-in = 采纳摩擦。
- **问卷3 = 三人中接地命中前置条件最好的样本**(会话在主线默认源、未物理丢失、真因在对话里),是最佳 dogfood 锚点;**但「前置条件好」≠「命中率高」,命中率本身是解法层假设 N=0,须 dogfood 分档实测**(见 §6 胶囊 C1)。
- 开放命门(不因 CC 暂时 2:1 消解):Cursor 主力默认关闭摩擦 + Cursor 接地质量非对称(漂白/对齐松/schema 非官方)。

## 4. ICP:收窄还是分层 → 分层(不收窄/不盲拓)

- **不收窄**:命题层 3/3 已证跨 ICP 普适,核心痛不限于高后果生产。
- **不盲拓**:freelancer 作为有效细分仍 N=1 假设;**且付费触发查无**——问卷3 对 decision-memory/考古品类零付费历史(问卷四Q7/问卷二Q8/问卷三Q3),只为主力 Claude 付费(180元/月+$20/月),与问卷1/2 同构。
- **分层动作**:
  - 定位/变现文案分两层。高后果生产(问卷1/2):挂「防¥3万级事故/防OOM/防产线停机」。低后果 freelancer(问卷3):**「防事故」话术无力**,潜在付费理由若成立只可能挂「客户代码隐私(接单生存线)」,而非防事故。
  - freelancer 特异需求(跨多项目切换捡上下文、跨项目区分汇报理由)记为拓宽候选,N=1 待验证。
  - 升人群门槛:≥3 freelancer 复现「多项目切换/客户隐私/汇报考古」**+ 真实付费信号**。
