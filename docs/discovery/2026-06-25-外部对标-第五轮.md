> 裁决一律过五门(护城河 / M0 红线 / 证据 N=1 / 竞品 / 北极星),给「可做 / 该拒 / 验证后再做」+ 红线内替代。

## 一、想法 2:文件树 + 新增/修改高亮 —— **可做(最该做)**,定位为「接地入口」非 diff 查看器

- **外部范式成熟、零-LLM 可做** `[证据:强]`:VS Code `gitDecoration.*` 一套标准颜色令牌覆盖 ≥10 种 git 状态(added/modified/deleted/renamed/staged-*/untracked/ignored/conflicting/submodule),同一映射在文件树与 SCM 双面复用(`https://code.visualstudio.com/api/references/theme-color`)。
- **底层是确定性、零-LLM 数据模型** `[证据:强]`:`git status --porcelain` 两字符 XY 码(X=index、Y=worktree)+ 固定单字母码(M/A/D/R/C/T/U),**porcelain v1「保证不向后不兼容、适合脚本解析」**(`https://git-scm.com/docs/git-status`)→ 机械映射到高亮,**契合零-LLM 接地哲学 + 零-build 单文件 vanilla-JS 红线**。
- **可访问性坑必须规避** `[证据:强]`:VS Code 单字母徽章有真实 onboarding 问题——用户 hover「U」看不到含义(microsoft/vscode#123103,原话「I had U in my tab, and I couldn't remember what U stood for」)。→ **默认给 tooltip/文字,不靠单字母+颜色**(颜色单独不可访问)。
- **护城河门校正**:纯「显示什么变了」是 GitLens/DeepWiki 地盘、**off-mission**(vibetrace 答 why 非 what)。正确定位 = **导航入口**:点变更文件 → `接地追问这个文件当初为什么改`,树成为通往 why 接地的可发现入口。
- **裁决:PASS(Now / 低成本 UX)**。落在已有 `report`/`console` web 视图;北极星门提醒:导航辅助,别指望直接动「处理率」。

## 二、想法 1 + 想法 3 合流:**只读收割命令历史/会话作接地素材;拒绝可执行 / OS 级全量**

研究把这两想法逼到同一红线内答案。

- **想法 1「捕获命令上下文」有价值** `[证据:强]`:VS Code shell integration 捕获命令检测+退出码、把历史 surface 进 Quick Pick(`https://code.visualstudio.com/docs/terminal/shell-integration`);Warp 以 Block(命令+stdout/stderr+退出码)为单位喂 AI、右键「Ask Warp AI」解释错误(`https://www.warp.dev/warp-ai`)。
- **但「可执行」终端是不可消除安全面 → 拒** `[证据:强]`:Cursor 一连串严重 CVE——CVE-2026-22708(空允许列表仍沙箱绕过+RCE)、GHSA-hf2x-r83r-qw5q(恶意网站间接提示注入+白名单绕过,Auto-Run 设 AllowList 仍任意命令执行),Cursor 官方自承允许列表是「best-effort guardrails rather than a hard security boundary」(`https://docs.cursor.com/en/account/agent-security`、`https://www.pillar.security/blog/the-agent-security-paradox-when-trusted-commands-in-cursor-become-attack-vectors`)。Warp 还把终端 IO 传出本机(Speedscale 抓包证实默认无本地处理,`https://speedscale.com/blog/what-the-warp-terminal-sends-home/`)。
- **想法 3 被动捕获有成熟市场** `[证据:强]`:Pieces LTM-2 OS 级捕获、存 9 个月;SpecStory 自动跨工具存 `.specstory/history/` Markdown、声称保留「reasoning, decisions, design tradeoffs」;Continue 默认本地落盘 `.continue/dev_data`(`https://pieces.app/...`、`https://docs.specstory.com/`、`https://docs.continue.dev/...`)。
- **但外部隐私默认值比营销松** `[证据:强,反向]`:强 local-first 表述被对抗验证**否决**(SpecStory「nothing leaves unless opt-in」0-3、Pieces「仅 LLM 出网」1-2)→ **vibetrace 的「落盘/出网前脱敏 + 只绑 127.0.0.1」反成差异化**,不能假设「被动捕获默认安全」。
- **裁决:验证后再做(红线内版)**:把已有会话捕获**扩展到只读 shell 命令历史(`~/.*_history`)+ Cursor 会话**,严格本地 + 脱敏 + 可关,作**接地素材**。**REJECT** OS 级「思考模式」全量捕获(撞 Pieces 腹地;N=1 用户明说**不为「记录」付费**——捕获是手段、接地 why 才是目的,别把「捕获更全」当卖点)。**REJECT 可执行内嵌终端**(M0 红线 + 安全 + 撞 Warp/Cursor 腹地)。

## 三、想法 4:多模型对抗审查(付费)—— **该拒为产品,留作内部手段**

- **本轮证据缺口** `[证据:无]`:5 个角度里这一项**无一条声明通过 3 票对抗验证**(0 confirmed)——是「未采到/未存活」,**非已证伪**。结合前几轮:review 工作流**撞 CodeRabbit/Greptile 腹地**(round2 已警示)。
- **护城河门(靠前几轮 + pm-brief,不靠本轮)**:多模型「判对错」正是 pm-brief 划清的边界——vibetrace **「不是替你审对错」**,是「找回 why」。付费墙在 dogfood 证明「真拦下一次理由丢失型踩坑」前 **NEVER 启动**。
- **裁决:REJECT 作付费 review 产品**(撞腹地 + 偏离护城河 + 付费过早)。**红线内替代 + 已在用**:「多模型对抗」模式本身有价值——作**接地质量的 dev-only 验证手段**(judge-panel 验真,正是本研究弧用的 adversarial-verify),而非面向用户的付费审查。已有 `review` 命令(零-LLM,给「这些行历史决策 + 真实引用」)才是该占的缝:**CodeRabbit 判对错,vibetrace 给 why**。

## 四、想法 5:code-to-course —— **已存在,不重复立项**

- `vibetrace course`(演进课程)**已实现**——pm 硬规则:别把已有能力当未建重复立项。
- **外部消费证据弱/存疑** `[证据:弱]`:DeepWiki 准确性受公开质疑(`https://finance.biggo.com/news/202508270142_DeepWiki_Accuracy_Concerns`),这类自动生成物消费度存疑;pm-brief:用户痛点是 why、非教程。
- **裁决:不新投入(Later/待评估)**。它是护城河数据的一个 LLM 消费视图,够用即可;加厚需先有 N=1 消费证据。

## 五、收口结论 + 诚实边界

1. **想法 2 最该做**,但定位成「点变更文件 → 接地 why」的导航入口,不是 diff 查看器(否则 off-mission 撞 GitLens/DeepWiki)。
2. **想法 1+3 收敛**成「只读收割命令历史/会话作接地素材,本地脱敏可关」;拒绝可执行内嵌终端 + OS 级全量捕获。
3. **想法 4 拒为付费产品**,但多模型对抗留作 dev-only 接地质量验证手段;面向用户仍是 `review` 给 why 而非判对错。
4. **想法 5 已有(course)**,不重复立项。

**诚实边界**:想法 4/5 是本轮最大证据缺口(0 条过对抗验证),对它俩的裁决靠**前几轮 + 主线代码 + pm-brief**,**非本轮 web 证据**;要对它俩下硬结论须另开针对性调研。隐私类声明多为厂商营销页,强 local-first 表述已被对抗验证否决,营销隐私默认值不可全信。
