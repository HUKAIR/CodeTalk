# G2 验证问卷:接地真实记录 vs AI 反推(信任位移 + 主力工具)

> **这是验证问卷,不是发现式问卷**(后者走 `README.md` 的 gap-analysis SOP)。它专测两件**能单独证伪产品**的事:
> ① 受访者**主力 AI 工具 + 决策对话能否找回**(命门假设,pm-brief 实验①,破 N=1);
> ② 看过「引真实记录 vs 纯 diff 反推」对照后,对 **AI 反推 why 的信任是否低于对引真实记录**(pm-brief 实验② = 6 分信任天花板)。
> 配套刺激素材见 `2026-06-29-护城河对照卡-真实记录vs反推.md`;本问卷已内嵌一份盲对照可直接发。

---

## 给受访者的话(可直接复制发出)

> 占你 5–8 分钟。**只问你真实经历过的具体事**,没有标准答案;诚实的「没遇到过 / 不信 / 没花过钱」对我们最有价值。涉及看两段文字的那题,凭直觉判断就好。

### A. 你与 AI 编码(只答真实发生过的)
- **A1.** 你现在主力用哪个 AI 编码工具?大概各占多少比例?(例:Claude Code 70% / Cursor 30%)
- **A2.** 上一次你想搞清「这段代码当初**为什么**这么写」是什么时候、哪个项目?当时你怎么找回的(翻 PR / 问人 / 问 AI / 猜 / 放弃)?最后找回了吗?
- **A3.** 那次的「为什么」里,有没有「为什么**没**用另一个方案 / 为什么先放着」这类?这种你**找得回**吗?
- **A4.** 你和 AI 当时的那段决策对话,**现在还找得到吗**?(工具里翻得到 / 已经没了 / 没去找过)
- **A5.** 最近一次因为「想不起当初为什么」而返工或踩坑:哪个项目?大概花了多少时间 / 代价?

### B. 两段「当初为什么」(同一个真实开源项目的同一个 commit,不告诉你它们各自怎么来的)

> 一次大改动(发布前修复 + 健壮性 + 几个新功能)。下面**甲、乙**是对「作者当初为什么这么改」的两份说法:

**版本甲**
> 作者把测试里的 mock 签名从 `*a` 改成 `*a, **k`,是为了未来兼容——被 mock 的函数后续可能传关键字参数,用 `**k` 吸收所有关键字省去反复改测试;否决了「为每个 mock 精确列参数」那种与实现过度耦合的做法。

**版本乙**
> 脱敏收口在单一入口(任何叙事来源落盘前都过一遍,杜绝未来新路径漏脱敏);course 崩溃只修调用签名、**不**套宽 try/except(TypeError 是代码 bug 非外部容错场景,吞了会掩盖真 bug);blame 复制一份解析而非耦合另一模块(两者各自演化);cli 拆分避免循环导入又不破既有测试契约。

- **B1.** 你更信哪份是作者**当初真实**的理由?给甲、乙**各打一个信任分(0–10)**。
- **B2.** 有没有哪份让你觉得「这是**编 / 猜**的」?具体哪句?
- **B3.** 如果你要据此判断「这段代码现在能不能动」,你会照哪份行动?
- **B4.**(看完下面这句再答)**版本甲是「只把 diff 给 AI、让它反推」;版本乙是「逐字引作者当初写下的真实记录」。** 知道这个后:你对「**AI 从 diff 反推 why**」打几分信任(0–10)?对「**工具逐字引真实记录**」打几分?

### C.(选答,但这题最有价值)用你自己的 commit 测
- **C1.** 愿意的话:挑你最近一个**你清楚记得为什么这么写**的 commit,把它的 diff 发我们(可自行脱敏)。我们**只把这份 diff 给 AI**、让它反推「当初为什么、否决了什么」,再把结果发回给你。
- **C2.** 你来判(你是作者 = 唯一知道真相的人):它命中了几成?**漏了什么?编了什么?**

### D. 价值(只问实际发生过的,不问意愿)
- **D1.** 如果「为什么」能逐字引真实记录、不靠反推,你估计 A2/A5 那种考古能给你**省多少时间**?(或:能不能让你**少踩那次坑**?)
- **D2.** 你过去为「找回 / 记录决策」这类能力**实际花过钱**吗?花在哪、多少?(没花过也是有效答案)

## 发放 + 记账注记(对内,**不发给受访者**)

**盲法(重要)**:B 段发出前**随机调换甲/乙顺序**,别让「乙总是真实」露馅。本版 key:**甲 = AI 反推、乙 = 真实记录**(取自 commit `4440324`)。换序后记下每位受访者拿到的实际对应。

**每位受访者要记的一行**:
- 主力工具 + 占比(A1);A2 是否找回 + 方式;A3 why-NOT 能否找回;**A4 决策对话是否还在**;A5 代价。
- **B1 甲/乙 信任分之差**;**B2 是否自发指出「反推那版」在编/漏**(自发指出 = 强信号);**B4「引真实记录」信任分 −「AI 反推」信任分**(← 这就是「破 6 分天花板」的核心数字)。
- C 段作者自判命中率 / 漏 / 编;D2 实际付费(金额、对象)。

**通过判据(到期对账,可复算)**:G2 通过 = **≥3 名受访者** 满足:① B4 中「引真实记录」信任分显著高于「AI 反推」信任分,且「AI 反推」基线 ≈6 或更低;**且** ② 能自发(B2)或在 C 段指出反推的编造/遗漏。任一不达 → 信任天花板未破,**不启付费墙**(对齐 ROADMAP:40 时序闸 G2)。

**证据纪律(承 vibetrace-pm + README SOP)**:
- A / C / D 走 **The Mom Test**——只采真实发生过的事,意见/意愿记 0;B 是**唯一允许的「看刺激」环节**(G2 本质是受控对比实验,非发现式)。
- **单受访者 ≠ 人群**:升人群结论门槛 = 同结论 ≥3 人复现;**绝不**把 N<3 写成「用户们/普遍」。
- **per-tool 命门**:A1 必抽。若 ≥3 人主力 Cursor 且 A4 多答「对话已没了」→ 整条会话价值链预警(pm-brief 实验①)。
- **偏置自知**:B 段对照取自我方仓、版本乙更具体(更长≈更可信是混淆项),故 **C 段(受访者自己的 commit、作者即真相)是更硬的版本**——能拉到 C 的样本权重更高。文案落点挂「找回 why / 防踩坑 / 省考古工时」,**不**挂「记录 / 留痕」。
