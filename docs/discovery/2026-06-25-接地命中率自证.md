## 结果(CodeTalk,78 commit)

| 指标 | 值 |
|---|---|
| commit 总数 | 78 |
| 有叙事(narrative) | 68(87.2%) |
| 有 Vibe-Decision 面包屑 | 43 |

| **可确定性接地回答 why(覆盖上限)** | **73/79 = 92.4%** |

> **同日修复 evidence=0**:`enrich` 此前覆盖补全用空会话对齐(`align([])`)丢了 evidence。
> 修法(零-LLM):`enrich` 改为扫真实会话对齐 + 新增 `enrich.backfill_evidence` 给已叙事的 commit
> 据对齐会话补 evidence 原话锚点(`cache.set_narrative_evidence`,只加 evidence、不动 LLM 的
> why/decisions,守 immutable;落库脱敏)。`vibetrace enrich --no-llm` 一跑 **evidence 0→68**,
> 护城河第三支柱「引真实会话原话」立起来。下方「诚实缺口」一节已消解。

## 解读

- **覆盖上限 92.3% 是强信号**:近全仓的 commit 都能零-LLM 确定性接地(叙事 why/决策 ∪ 面包屑)回答「当初为什么」——**green-light** review-entry 扩面与 grounding-eval(撞腹地/高成本项的命中率前置已过)。6 个未覆盖的多是机械/纯文档 commit。
- **诚实缺口:evidence 会话原话锚点 = 0**。护城河三支柱之一「引真实**会话原话**」在本仓当前**零覆盖**:① 本会话的 `enrich` 覆盖补全用空会话对齐(align([]))→ 不带 evidence;② 早期 digest 富集的叙事也未留存 evidence 锚点。**当前接地靠的是 commit 叙事决策 + Vibe-Decision 面包屑,不是会话原话**。对外文案据实:讲「引真实 commit/决策」可,讲「引真实会话原话」在本仓暂无据。
- **要补 evidence 覆盖**:需在**会话仍在**时跑 digest(`sources` 含 claude/cursor)富集,让 `_evidence` 收割 session 原话锚点;或给 `enrich` 加可选会话对齐(当前是 align([]))。列 Next。

## 闸门裁决(对下游项)
- **review-entry 扩面 / grounding-eval**:命中率前置 ✅ 过(92.3%)→ 可启动(grounding-eval 仍按对抗复跑结论走「逐字命中片段高亮」,不做逐句 n-gram 标签)。
- **本地 LLM / ide-ext**:命中率不是其阻塞项,按各自取舍。
- **未决**:evidence 会话原话覆盖(0%)是真缺口 → 补「会话在场时富集」或「enrich 可选会话对齐」,再复跑本脚本看 evidence 列是否非零。
