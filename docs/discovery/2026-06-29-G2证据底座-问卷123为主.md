## 按 G2 假设盘点(问卷123 为主)

**① 信任天花板(信不信 AI 反推 why)**
- 问卷1:章六 Q2 对「AI 重解释 why」**信任打 6 分**——`有证据(N=1)`,**这是「6 分」的唯一来源,不是人群锚**。
- 问卷2:把代码贴回 Claude、它编「内存优化」,真因是「流式响应」,**信了损失 2h、方向走偏**——`有证据(N=1,行为级,比打分更硬)`。
- 问卷3:「Continue 经常编造解释」「Claude Code 至少不会瞎编」——`有证据(N=1,定性二元)`。
- 三人**方向一致**(不信 AI 反推 why),但**只有问卷1有分数、问卷2/3 查无打分** → G2 验的是**差值方向能否 ≥3 人复现**,非「破 6 分」。问卷1 T11/T12 同时校准:**认可 AI 加速 what、只不信反推 why**(别测成全面不信 AI)。

**② why-loss / 决策上下文痛**(最强,已达人群)
- **3/3 人群结论**:why > what 更难找回、来由不清返工(代价分层:问卷1 ¥3 万 / 问卷2 小时级 / 问卷3 分钟级,不可等同)。
- 问卷1 真实事故:`Math.floor` 改 `round`、**超发 30 万积分、成本超 3 万**——「理由丢失型踩坑」的活样本。
- 问卷2:CLAHE 2.0「我没写为什么、AI 也没问就照搬」=**压根没记**(写时捕获要打的痛)。

**③ AI 编造理由**
- 问卷2「内存优化」编造(②里那条,最硬行为锚)+ 问卷3「Continue 经常编造」→ `竞品/态度层 3/3`,但「编造致**可量化损失**」硬实证仍 `N=1(仅问卷2)`。

**④ 决策对话 / 历史找不回**(命门,**按主力工具分叉**)
- 问卷1(Cursor)大部分没了 → **生存级错位**:vibetrace 主线读 Claude Code,而他的 why 几乎全在 Cursor。
- 问卷2(Claude Code)有的没了;问卷3(Claude Code)未丢、3min 找回。
- → 「会话能否找回」不是二分,是**载体状态 × 主力工具**;这是 pm-brief 实验①(会话价值链)的真变量。

**⑤ 「引真实记录 比 AI 反推更可信」的差值**(vibetrace 独有命题)
- 问卷123 **查无**直接测过这个差值(它们问的是「信不信 AI 反推」,没有 A/B 对照)。
- → **这条只能靠 vibetrace 自己的对照卡 + C 段(受访者自有 commit、作者即真相)验**,公开数据也几乎不会有(见下二手节)。

## 公开二手佐证(全网检索 · 人群尺度 · 仅佐证「问题存在」)
> ⚠️ **web 检索初采,数字外引(pm-brief / 落地页)前须回原始来源逐一核对**(守「对外数字必附可复算口径」红线)。强度档:survey(强)> 一手 blog/bug(中)> 论坛轶事/厂商(弱)。仓内 `外部对标-第二/六轮` 已有的(MSR 2026 AI-review −23pp、Trust-Score/幻觉检测)不重复。

**① 信任天花板 —— 二手最硬**
- **SO 2025**(子样本 N≈31k–33k):highly-trust 仅 3.1%;distrust 45.7% > trust 32.7%;对准确性信任 ~40%(2024)→29%(2025);采用率却升到 84%(信任↓采用↑的悖论)。`[survey 强]` survey.stackoverflow.co/2025/ai
- **SO 2024**:遇困难头号问题逐字「Don't trust the output」66.2%。`[survey 强]`
- **DORA 2024**:39% 对 GenAI 输出质量只「a little / not at all」信任(页未披露 N)。`[survey 中强]` dora.dev/research/ai/trust-in-ai
- **Sonar State of Code 2025**(N>1,100):96%「不完全信任」AI 代码、53%「appears correct but unreliable」(媒体二手转述)。`[survey 强/转述中]`
→ 据此把 N=1 的「6 分」从主证据降为「我的深访也复现了人群结论」的佐证。

**② why-loss / 决策上下文痛**
- **Atlassian/Wakefield**(N=3,500、6 国):开发者头号时间杀手「finding information」(口径偏宽=泛化找信息,非专指 why-loss,外推谨慎)。`[survey 强,口径宽]`
- 学术行为观测:开发者 58–70% 时间在「理解现有代码」(Feitelson 2023 引 Minelli/Xia)。`[survey 强,上位证据]` arxiv 2410.21990
- 一手轶事(精准命中):Claude Code compaction 丢上下文,原话「the 'why' behind choices — gone」。`[blog 中]` dev.to/gonewx

**③ AI 编造理由(机制证据强、命中数字弱)**
- arxiv 2510.25662《User Misconceptions of LLM…Programming Assistants》:把 LLM 自我解释当 why 是误解,self-explanation 不忠实、post-hoc。`[学术 中强,机制对口]`
- arxiv 2312.09126:解释「may be hallucinated」,知识缺口处「fill the gap with something plausible」。`[学术 中强]`
- 行为实验:AI 错误建议下开发者仍 34% switch rate = overtrust(行为非观点)。`[实验 中]` arxiv 2508.08158

**④ 决策对话/历史找不回(命门浅层 · 只轶事密度、无大样本)**
- 官方 bug(最硬客观证据):Anthropic repo issue #45076「session history… silently lost」(跨平台);Cursor 论坛「Lost All Chat History」厂商承认 beta「rare condition」。`[bug/forum 一手,规模不可计]`
- 自建补救(行为 > 抱怨):开发者把 Claude Code 会话归档进 SQLite,因 JSONL「just gone」。`[blog 中]` dev.to/kuroko1t
- **公开真空**:无任何带 N 的调查专测「AI 编码对话历史丢失」。

**⑤ 接地 vs 反推「差值」(vibetrace 命门)—— 公开几乎为零**
- 间接旁证:SO 2025 75.3%「不信 AI 答案时会转去问真人」(意向、且是「转向人」非「转向 git 原话」,降档);arxiv 2603.17150《Intent Formalization: A Grand Challenge》把「intent 无法可靠从代码反推」立为公开难题(仅 abstract)。`[中]`
- **真相:无任何公开数据直接量化「翻 git/PR 原话 vs LLM 重述 why」的可信度差值——这一格只能自己造数据**(对照卡 + C 段)。

## 结论(诚实)
- **二手能坐实**:① 信任天花板、② why-loss 痛、③ AI 会编造理由——均有带 N 官方调查(SO/DORA/Atlassian/Sonar),样本比 N=3 大四个数量级。**省掉「证明问题存在」的活,直接引用即可**(N=1 的「6 分」降为佐证)。
- **二手替代不了、必须自己验**:**⑤ 差值(引真实 > 反推)** 与 **④ per-tool 命门(决策对话在哪 / 还在不在)**——公开数据不测这两条。
- **🚩 红线警告**:别把 survey 的「普遍不信 AI」当成「我的差值已验证」——前者是市场背景,后者是产品有效性;把背景当验证 = 没地基盖楼(pm-brief 第 14 行已钉死)。
- **省力路线**:二手坐实 ①②③(引用,零访谈)+ 自己只跑 **1–3 人**验 ④⑤(甚至先并进问卷 C 段自有-commit 顶上,真人只做最终背书),而非从零证明痛点。`D 实付=0 不否决`;信任=G2、付费=G3 脱钩。
