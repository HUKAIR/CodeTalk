# vibetrace 发现式研究 · 六主题专家问卷合集

> 一份面向资深 / 重度 AI 编码者的发现式研究合集:六个主题,统一走 The Mom Test —— 线上书面、小样本、可深挖。全程只问"你自己亲历的上一次具体事件",不问看法、不问意愿、不让排序功能。

---

## 一、共享方法论(所有六章通用,各章不再重复)

下面这套口径对六份问卷一律适用。各章只保留**该主题独有**的研究目标、反陷阱、独有题与判定维度;通用部分一律回到这里。

**形态。** 线上书面问卷,受访者异步自助作答。没有现场追问,所有追问内嵌成"必填例子栏 + 子提示";第二轮另起一轮短问。

**对象。** 专业 / 资深开发者,多为重度 AI 编码者(Cursor / Copilot / Claude Code 等),本人即一线写代码 / 一线决策者优先。

**样本。** 小 N(约 6–8 人),每份都求深,不求广;每份都预留第二轮,对答得实的人再钻一层。

**方法。** The Mom Test —— 全程只取受访者**过去真实发生过、亲手做过**的行为,不问意见、不问接受度、不问付费意愿,不给功能清单让排序。

**专家防 pundit 铁律(本批受访者最大污染源)。** 专业开发者天然爱切换成"评论员"口吻答"最佳实践 / 一般来说 / 好工程师应该 / 用户一般会……"。处理规则统一为:

1. 任一题答出"最佳实践 / 一般来说 / 用户应该 / 大家都……"且报不出具体一次 → 该题记 **0 分**,在该受访者卡片累计一次 pundit-flag。
2. 第二轮强制把他拽回"**你本人上一次**":"先别谈一般情况,就那一次,你本人当时点了什么 / 删了什么 / 做了什么?"
3. pundit-flag 累计 ≥ 3 次,整份样本可信度降级。
4. "完美人设"(如"我 commit 一向写得很细""我从不把敏感东西给云端")同样标记为可疑,需用具体回忆题反向核验。

**真实性硬门槛(过线才计分)。** 一条经历要被采信,必须报得出 **具体项目 + 具体文件 / 代码段(或 PR / bug) + 大致时间**;留痕 / 付费类题还需贴得出原话、截图或具体金额。三者缺一,该条整体降为"线索 / 背景",不进任何评分维度。"我一向写得很细 / 这类工具值得投入"这类无锚点表述,直接记 0。

**意见 / 意愿一律记 0。** 接受度、付费意愿、对别人 / 对行业的判断,全部不进任何维度。所有"钱"的问题只问"过去实际花没花、花在哪、多少",绝不问"愿不愿意花"。诚实的负面信号("翻遍也没找到""AI 对话早没了""从没花过钱""排不进前五")当作成功记录,而非失败 —— 它们是判断"要不要做这个产品"的最高价值数据。

**通用评分口径(频率 × 后果 × 已投入,乘法)。** 每个维度按"能否落到一次具体事件"三档计:**2 = 工具名 / 项目名 + 时间 + 动作俱全;1 = 有事件但缺要素(时间模糊 / 动作含糊);0 = 仅态度 / 意见 / 意愿 / 二手**。综合时取乘法而非加法 —— 任一维为 0 即接近出局,体现"高频但无痛""听起来好但从没做""没人投入"都不该优先。归因校验两条通用规则:**自发进榜的痛 ×1.3**(开场空白题里自己捞出来的),**追问才冒出的痛 ×0.8**(被诱导出来的不可信)。

**小 N 分析法。** 这是深度个案研究,不是统计抽样。不报"X% 的人怎样",而是把每个第一手个案当一条完整因果链读(为什么做 / 没做 → 后来被不被需要 → 缺了它付了什么代价),逐人建一条时间线,横向找**重复出现的同一类触发 / 同一类断档**。矛盾或离群个案优先深挖第二轮,不取平均抹平 —— 小 N 下离群点常含最强洞察。

**通用招募与投放。** 冷外联**首条只发开场题**降低回复门槛,对方回了再追问;**followup 按对方答案择一两条追**,不要清单式逼问;异步看不到表情 / 犹豫,务必盯"能否报出具体项目 + 文件 + 时间"作硬门槛,含糊即中性追问"最近具体哪一次?"(别替他举例);对方主动夸功能 / 提需求时**不接、不确认,拉回"上次具体那次你做了啥"**;先发 3 人小批跑通问法,看哪题总收不到具体例子,迭代后再发剩余。

## 第 1 章 · 理解债痛点验证

> **本章独有研究目标:** 在动任何方案之前,先验证"开发者回头看不懂自己或 AI 写的代码、想不起当初为什么这么定"这个问题**到底真不真实存在、有多痛、多高频**。这是整条研究链的地基 —— 问题不成立,后面五章都白做。

**本章反陷阱(独有部分):**
- **不诱导强版本痛。** 不用"完全想不起来""根本看不懂"这类强度词诱他承认重症;一律用中性的"卡了一下",让强度由他自己定。
- **不喂菜单。** 不报"是不是翻了 git log / 问 AI"这种选项;开放问"做了哪几步",让他自己说出用了什么,沉默就等他补。
- **不预设坏结果。** 不问"是不是引了 bug / 是不是返工了";开放问"后来怎么发展",允许答案是"没造成什么事"。
- **频率不靠拍脑袋。** 不问"大概多久遇到一次";让他枚举具体实例,频率由实例自然反推。
- **末题不替他塞结论。** 最后那道"最近头疼的开发问题"别把本痛点替他塞进去,让本痛点是否**自发进榜**自己说话。

### 开放题(8 道,暖场 → 卡壳定位 → 频率 → 后果 → AI 失忆 → 投入)

**Q1 · 暖场:你最近在写什么**
最近一周你具体在写什么项目 / 什么功能?昨天或今天主要在 AI 辅助下做了哪段?
〔必填:项目 / 功能 + 用的 AI 工具〕
> 子提示:这个项目你做多久了?现在主要用哪个 AI 工具在写(Cursor / Copilot / Claude Code …)?最近一次让 AI 连着写一大段代码,是在做什么?
>
> 〔此题纯暖场:确认他确实是重度 AI 编码者、有活跃真实仓库可锚。别急着把话题引向"看不懂";听他是否自然冒出"回头看 / 改不动 / 忘了" —— 主动冒出来即强信号。〕

**Q2 · 最近一次回看自己代码、读着卡了一下**
最近一次你回去看自己仓库里某段代码、读的时候卡了一下,是什么时候?那是哪段代码?
〔必填:那段代码大概干嘛的 + 大概何时〕
> 子提示:那段代码大概多久以前写的?是你手写的还是 AI 写的?当时你为什么要回去看它 —— 加功能还是修 bug?你怎么意识到自己卡住的 —— 报错了,还是读着读着没头绪?
>
> 〔能立刻报出具体文件 / 模块 / 时间(如"上周那个 webhook 重试")=痛真实;支吾"应该有吧""我都写注释"=没真实发生或不显眼,如实记弱信号。〕

**Q3 · 那次你具体做了哪几步去搞明白**
上一次发生这种卡住的时候,你具体做了哪几步去把它搞明白?能按当时的顺序说一下吗?
〔必填:按真实顺序的动作链 + 大概何时〕
> 子提示:这中间最花时间的是哪一步?最后是看懂了,还是绕过去 / 删了重写?那次大概花了你多久 —— 当场十分钟,还是拖了好几回?
>
> 〔能描述真实动作链(git blame→翻 AI 旧对话翻不到→整段重写)且明显费劲=痛真实有应对;"看一眼就懂""没花功夫"=代价低、不痛。〕

**Q4 · 枚举近两个月还有哪几次**
过去一两个月里,这种回头看不懂自己或 AI 写的代码的情况,你还记得的有哪几次?分别是哪个项目、当时在做什么?
〔必填:列 2 次以上,各自项目 + 当时在做什么〕
> 子提示:这几次里哪一次最头疼?为什么是那次?是某类代码(不熟的框架、复杂状态逻辑)特别容易,还是哪儿都可能?有没有反过来 —— 某段 AI 写的隔很久回看依然一目了然?那种和卡壳的差在哪?
>
> 〔列得出 2 次以上彼此独立、细节不同的事件=高频真实;只给笼统频率词列不出事件、"多次"其实是同一件反复讲、或坦白"其实很少基本看得懂"=频率不足。〕

**Q5 · 最头疼那次后来怎么发展**
你刚说最头疼那一次,卡住之后那件事后来实际怎么发展的?
〔必填:那次的真实后续〕
> 子提示:那次大概搭进去多少时间,或者影响到了什么别的事(上线延期、功能回退、白干的活)?如果出了问题,是你自己发现的还是别人 / 用户发现的?事后你有没有改什么做法来防它再发生?改了的话坚持下来了吗?
>
> 〔说得出具体下游代价("多花一下午""上线炸了回滚""整块重写")=痛够深;后果只是"多看几分钟""读两遍就懂"=轻微摩擦,如实记不够痛。〕

**Q6 · AI 决策失忆:想不起 AI 当初为什么那么定**
最近一次你想回忆 AI 当初替你选了某个库或某种实现、为什么那么定,却一时想不起来,是什么情况?
〔必填:那个决定大概是什么 + 大概何时〕
> 子提示:那个决定是 AI 自己定的,还是你当时点头了后来忘了为什么?你后来怎么查回当初为什么那么定的?查得到吗?"想不起代码怎么写的"和"想不起当初为什么那么定的",你更常碰到哪个?
>
> 〔举得出一次决策失忆的具体事件并说清怎么查、查没查到=AI 决策失忆这一支成立;"决定都是我自己拍的记得很清楚""没遇过"=对此人这一面不成立,记证伪。〕

**Q7 · 最近一次特意留点东西给以后的自己**
你写代码的时候,最近一次特意留下点东西好让以后的自己看懂,是什么时候、留了什么?
〔必填:留了什么形式 + 大概何时〕
> 子提示:这习惯坚持下来了吗,还是写着写着就懒了?上次断在哪?回头你真去翻过自己留的这些吗?有没有翻了发现没记 / 记得没用?你大概花多少时间在这种"为了以后看懂"的记录上?
>
> 〔有自建凑合法(手写详细 commit、专门存 AI 对话、维护 decision 笔记)并承认费时 / 常断档 / 翻起来不好用=痛到值得自救;"没特意做""代码即文档不用记"=投入为零。〕

**Q8 · 最近一次为此掏过钱 + 真实烦恼清单收尾**
为了理解旧代码或找回当时的上下文这类事,你最近一次掏钱(订阅、买工具)是什么时候、买的什么?
〔必填:具体工具 / 订阅 + 大概何时;从没花过请直说"没花过"〕
> 子提示:这个是专门冲着"看懂旧代码 / 记住 AI 干了啥"去用的,还是顺带?有没有为这个目的试过某工具、用一阵又弃了的?为什么弃?**最近这一两周真正占掉你时间、让你头疼的开发问题有哪些?**
>
> 〔已为相关能力付过费或认真试用过专门工具(哪怕弃用),说得出花多少、为什么=最硬信号;完全没花过钱也没找过工具、"用不着白送也无所谓"=痛存在但不足以驱动购买。末题让他列真实烦恼,看本痛点是否自发进榜。〕

### 第二轮深挖
> 一次只发 1–2 题,拿到带具体例子的回答再追问,别一口气甩 8 题。受访者答得泛("经常吧""一般还行")时,回一句中性"能想到具体最近的一次吗?",不替他举例。把"证伪信号"("看一眼就懂""没花过钱""排不进前五")当成功记录。

- 承 Q2 / Q3:把那次最费劲的动作链复原一遍,哪一步最堵?那次最后是真懂了还是带着不确定改了?
- 承 Q4 / Q5:你枚举的几次里,有没有一次后来真出了下游事故?完整复盘一次。
- 承 Q6:那次决策失忆,你最后是怎么收场的(照搬 / 重写 / 弃)?当时的 AI 对话还找得回吗?
- 承 Q7:你留的那东西,后来真有人(或你自己)回来翻过吗?坚持的习惯上次断在哪一刻?

### 本章判定 / 优先级维度
> 总口径见共享方法论。本章问题成立的核心判据:

| 维度 | 取证题 | 强信号(2) |
|---|---|---|
| 卡壳真实性 & 强度 | Q2 | 立报具体文件 + 时间 + 中性"卡了一下"由他升级为真痛 |
| 应对成本 | Q3 | 真实多步动作链 + 明显费劲 + 耗时数字 |
| 频率(实例反推) | Q4 | 列得出 2 次以上独立事件 |
| 未做好的代价 | Q5 | 具体下游代价(返工 / 回滚 / 重写) |
| AI 决策失忆(差异化) | Q6 | 一次具体失忆 + 查没查回,单独标记不被稀释 |
| 已投入(自救) | Q7 | 自建凑合法 + 承认费时 / 断档 |
| 已花的钱(最硬) | Q8 | 真实付费 / 弃用工具 + 本痛点自发进榜 |

## 第 2 章 · 采纳飞轮 / 留痕障碍

> **本章独有研究目标:** 验证 vibetrace 护城河赖以成立的那条行为飞轮 ——**"记下为什么"→ 事后"回头能找回"→ 被复用** —— 在资深开发者身上**到底会不会真的转起来**。这是头号风险:问题(第 1 章)就算成立,只要"记"或"回看"这两端有一端根本不发生,产品的现成需求就不存在。

**本章引言(直接发给对方):**
> 嗨,占你 15 分钟。我在研究**资深开发者写代码时,到底有没有顺手记下"为什么这么写",以及事后回头还能不能找回当初的缘由** —— 纯粹想了解你真实发生过的具体经历,不是问你觉得"应该怎么做"。
> 三个小请求:① 每题都锚在"你自己上一次"那件具体的事(哪个项目、哪段代码、大概什么时候);说不出就直接写"想不起具体的一次"(这也是有效信息)。② 能贴原文 / 截图就贴(commit 正文、注释、TODO、PR 描述、AI 对话片段),脱敏随意。③ 请讲你亲手做的,别替团队代答;旁观的请标一句"这是我看别人的"。记得越糟、坑踩得越实,对我越有价值。

**本章反陷阱(独有部分):** 本主题是"专家最易答应然"的重灾区 —— 凡回答滑向"应该 / 一般来说 / 大家都……"且报不出具体一次,该题记 0。Q5(明知该记却没记)用受访者自己的话先答"是什么把你拦住了",之后才允许对照清单,严禁先喂"是赶时间 / 觉得没人看"等选项。

### 资格筛选(1 题)
**S1.** 过去 3 个月里,你亲手提交过代码(commit / PR / 自己的项目)的那个仓库,最近一个是哪个、大概在做什么?最近一周你大概在它上面动了几次手?
〔必填:项目名或简述 + 最近一次动手大概何时〕
> 〔区分"第一手在写代码的人"与"主要在管理 / 旁观的人"。近 3 个月几乎没亲手提交请如实说明,后续题对你多半二手,我会据此降权。〕

### 开放题(8 道)

**Q1 · 你最近几条 commit 的正文,实际长什么样**
把你最近 3–5 条真实 commit 想一下(看一眼 `git log` 最准):有几条正文是空的、有几条在正文里写了"为什么"这么改?其中最近一条你确实解释了取舍 / 原因的,是哪条、大致写了什么?
〔必填:项目 + 大概何时 + 那条带"为什么"的正文原话(脱敏后照贴)〕
> 可以多说一点:你写了正文那几条,是因为那次特别难 / 特别绕,还是有别的具体触发?正文空着的那几条,半年后有人问"为什么这么改",你现在还答得上来吗?

**Q2 · 上一次你写完一段"不太显然"的代码后,把"为什么"记了下来**
回到你正在动的那个项目 —— 上一次你写完一段不那么一眼能看懂的实现后,把"为什么这么写"记到了某个地方,是什么时候?记在了哪(commit 正文 / 注释 / TODO / 个人笔记 / 截图存的 AI 对话 / 群里发一句 / 哪都没记)?
〔必填:那段代码大概干嘛 + 大概何时 + 你当时记的那一句原话(若没记,直接写"没记")〕
> 可以多说一点:这句是当场顺手写的,还是事后补的?花了多久?那段代码里有没有 AI 替你拿了主意的部分?那部分你记没记它"为什么那么选"?

**Q3 · 上一次你为了搞懂旧代码"当初为什么那样写"而回头去翻**
最近一次你回头去看自己以前写的代码、想弄清它当初为什么那样写,是哪天、哪段代码?当时你怎么把答案找回来的(翻 git log / 翻 PR / 翻聊天记录 / 问人 / 直接读代码猜 / 干脆重写)?
〔必填:哪段代码或哪个 bug + 大概哪天〕
> 可以多说一点:你最后找到当初真实原因了吗,还是只能靠猜 / 放弃?从开始找到找到(或放弃)花了多久?那段代码是你自己写的、同事写的、还是 AI 生成你当时没细看的?

**Q4 · 上一次你接手一段"看不懂当初为什么这么写"的 AI 生成代码**
最近一次你接手 / 继续别人(或半年前的自己)用 AI 大段生成的代码,看不懂它当初为什么那么写 —— 是哪次?你怎么处理的(硬读 / 问原作者 / 翻当时的 AI 对话 / 直接推倒重写 / 带着疑惑先用着)?
〔必填:哪个项目 / 哪块功能 + 大概何时〕
> 可以多说一点:当时那段代码对应的 AI 对话或 prompt,还找得回来吗?找得到你真去翻了吗?你最后是搞懂了它为什么那么写,还是带着不确定就改 / 用了?

**Q5 · 上一次你"明知该记,却还是没记下来"**
上一次你心里清楚"这个决定以后可能会有人(包括我自己)回来问为什么",结果你还是没把它记下来 —— 是哪一次?用你自己的话说说当时是什么把你拦住了。
〔必填:那个决定大概是什么 + 大概何时〕
> 可以多说一点:(在你用自己的话答完之后,如果想对照一下:是赶时间、觉得记了也没人看、不知道记哪、还是觉得自己肯定记得?—— 只挑真符合你那次的说,别凑。)后来这事吃过亏吗?有没有人真回来问起过这个决定?发生了什么?

**Q6 · 上一次你专门标了一句"这里有风险 / 临时方案 / 以后要回来验证"**
最近一次你在 commit 正文、注释或聊天里专门标了一句"这块有风险 / 这是临时方案 / 这里以后要回来验证"(比一句光秃秃的 FIXME 更具体,说了为什么)—— 是哪次、原话是什么?后来你真回来处理它了吗,还是它就一直留在那?
〔必填:哪段代码 + 大概何时 + 那句标注的原话〕
> 可以多说一点:你后来是怎么(有没有)被提醒回来看它的?还是纯靠后来踩了雷才想起?那条标注最后被处理了、过期了、还是被无视了?你最近有没有"标了一堆 TODO / FIXME 最后全成摆设"的经历?

**Q7 · 你曾认真维持过某种"留痕"习惯,后来半途放掉了**
你曾经认真维持过某种留痕习惯(写决策日志 / 详细 commit / 工程笔记 / 存 AI 对话 / changelog),后来半途放掉了吗?是哪一个、坚持了多久、大概什么时候、因为哪件具体的事就不写了?
〔必填:哪种习惯 + 坚持时长 + 放弃大概何时〕
> 可以多说一点:放弃前的"最后一根稻草"是哪件具体的事?放掉之后,你有没有因此漏掉 / 找不回某个重要决定?现在还有它的残留吗(偶尔还写几句),还是彻底不写了?

**Q8 · 上一次你为"记录 / 回看代码与决策"真花过钱或装过工具**
你过去为"记录、回看代码与决策"这件事真金白银花过钱、或专门装过工具吗(付费 wiki / Notion / Linear、IDE 插件、git 工具、AI 对话归档工具…)?最近一次是什么?当时是被哪件具体的事推着去弄的?现在还在用吗?
〔必填:具体工具名 + 触发那件事大概何时 + 现在还用不用〕
> 可以多说一点:装 / 买之前,是哪次踩坑让你觉得"得弄个东西来解决"?用了一阵后,当初那个"看不懂当初为什么"的问题真被解决了吗?如果现在不用了,是什么时候、因为什么停的?
>
> 〔本题只问你过去真实发生过的花费 / 安装,不是问"愿不愿意付费"。从没为此花过任何成本也请直说 —— 这同样是有效答案。〕

### 第二轮深挖(按首轮答案择题追问)
> 第二轮只对首轮能报出具体项目 + 文件 / 代码 + 时间的受访者发起。

- 承 Q1(commit 基线):把那条带"为什么"的正文和紧挨的另一条空正文放一起 —— 这两次改动有什么不同,导致一次写了、一次没写?你 / 团队对 commit 正文有约定吗?实际执行得怎么样、谁在查?
- 承 Q2 / Q6(留痕):你记的那句话,后来真有人(或你自己)回来用到 / 翻过吗?怎么知道被用到的?从没被回看过,你觉得它白记了吗?

- 承 Q4(AI 黑箱):经过那一次,你对"自己用 AI 生成代码时怎么留痕"有没有真的改过什么做法?(只说你确实改了的。)
- 【记了又不翻 · 飞轮另一端的断点】上一次你记了点东西结果后来自己都没再翻、或根本找不着 —— 是哪次?记在哪了?为什么没用上(忘了在哪 / 过时了 / 搜不到 / 懒得翻)?深问:你记的地方和后来找它的地方是不是不同一个?这种"记了等于没记"多久发生一次?
- 【协作侧追溯】最近一次在 review 别人 PR、或读同事代码时,你卡在"这里当初为什么这么做",不得不去问作者或翻历史 —— 哪次?最后怎么拿到那个"为什么"的?深问:你是先自己翻了 git / PR / 聊天记录多久无果才问人,还是直接戳人?作者当时还记得吗,还是连他自己也忘了?
- 【被要求复述当初缘由】最近一次你向别人(同事 / 面试 / 复盘 / 写总结)讲清"当初为什么这么设计 / 这么改"—— 哪次?讲的时候你能直接调出当时依据,还是只能凭记忆重建、甚至后来发现记错了?深问:临时狂翻现找,还是手头早有现成记录?这种场合多久遇到一次?
- 承 Q7(习惯放弃):如果当初让你放弃的那个具体摩擦不存在,你觉得你今天还在写吗?讲那一次拐点前后的真实变化,别讲一般规律。

### 本章判定 / 优先级维度(飞轮判定)
> 总口径见共享方法论。

| 维度 | 它在测什么 | 取证题 | 强信号(2) |
|---|---|---|---|
| D1 留痕基线·已发生 | "记关键决策 / 为什么"现在到底有没有真发生、落在什么介质 | Q1, Q2 | 贴出真实原话 + 说清介质与场景 |
| D2 回看·已发生 | "为搞懂旧代码缘由而回头翻查"真实频次与触发 | Q3(+R2 记了不翻) | 报得出具体哪天 / 哪段 + 真实翻查路径与耗时 |
| D3 AI 决策黑箱痛 | AI 生成代码"看不懂当初为什么"的真实遭遇与处理 | Q4 | 具体一次 + 真实处理动作 + 结果(尤其"AI 对话翻不回") |
| D4 摩擦点 / 习惯杀手 | 什么具体地阻断了"记"与"持续记" | Q5, Q7 | 具体事件 + 真实阻力 / 导火索(+事后代价) |
| D5 闭环率 / 待验证项归宿 | 主动标的风险 / 临时方案,有没有真被回来处理(对应 Vibe-Watch 行为) | Q6 | 贴得出原话 + 诚实交代后续(含"再没回去过") |

## 第 3 章 · 信任与隐私边界

> **本章独有研究目标:** 判定隐私是不是 vibetrace 的**采纳闸门**,以及本主题命门 ——**"工具读我跟 AI 的对话"是否比"读我的代码"更触雷**(这条线决定产品的隐私默认值)。下发给受访者时只发"引言 + 资格筛选 + 开放题"三部分,分析块不外露。

**本章引言(直接发给对方):**
> 你好,占用你大概 15 分钟。我在研究开发者在使用工具时对"代码 / 跟 AI 的对话会离开本机"这件事的**真实做法** —— 注意,是做法,不是看法。所以每题我都只想听**你自己亲身经历的那一次具体的事**:哪个工具、哪个项目、大概什么时候、你当时实际怎么做的。我不要你对行业、对最佳实践、或对"别人 / 一般用户该怎么办"的判断。某题真想不起任何一次亲历的,直接写"没遇到过"。越具体越好(工具名 / 文件 / 大致时间最佳),带「必填例子」栏的请务必填。

**本章反陷阱(独有部分):**"完美人设"是本主题特有坑 —— "我从不把敏感东西给云端"同样标记为可疑,需用 Q4 / Q5 子提示的具体回忆反向核验(例:反问他最近 3 次 AI 对话里有没有出现内部域名 / 客户名 / 报错路径,校验"无所谓"是不是真的)。

### 资格筛选(1 题)
**Q0.** 过去半年,你本人有没有亲手装过 / 配过 / 退过会读取你本机代码或你与 AI 对话内容的开发工具(IDE 插件、AI 编码助手、代码托管、CI、遥测 / 分析类都算)?
〔必填:写出其中 1 个的名字 + 你大概什么时候动手的〕
- A. 有,是我本人操作的 → 第一手,继续全卷
- B. 有,但主要是别人(团队 / 运维)弄的,我旁观 / 听说 → 旁观者,后续默认二手降权
- C. 没有 → 本卷可能不适合你
> 只有能报出"具体工具 + 大致时间"的 A 类计合格第一手样本;B / C 仅作背景。

### 开放题(7 题)
> 排序:先用一次低门槛"拒绝授权"热身,再进"放弃 / 卸载"完整事件,然后是本地化的"已投入"代价,接着触命门"读对话 vs 读代码",再到"为省事破例"的真实权衡与补救,最后落到组织闸门与一次真实(准)事故的行为改变。

**Q1.〔热身〕拒绝过的那一项授权**
最近一次,某个开发或 AI 工具弹出权限 / 授权请求(要读你的代码库、剪贴板、整个 home 目录、浏览器历史,或要连你的 GitHub 账号),你看完后拒了某一项、或干脆没装 —— 是哪个工具、它要什么、你拒了什么?
〔必填:工具名 + 它索要的那项权限 + 你大概什么时候遇到的〕
> 子提示 a:它索要的哪一项让你停下来了?你当时怎么判断"这项不能给"的?
> 子提示 b:拒了那项之后,这工具你还用得了吗,还是因此就没用成?

**Q2. 上一次因隐私顾虑而放弃 / 卸载 / 退订**
回忆最近一次:你因为"数据 / 代码 / 对话会离开本机"或隐私顾虑,而放弃试用、卸载或退订某个开发工具,是哪一次?写出工具名、大概什么时候、当时你具体在担心什么。
〔必填:工具名 + 大致时间 + 当时具体担心的那件事〕
> 子提示 a:让你警觉的那个具体瞬间是什么 —— 某条权限请求、某份条款里的某句话、一次数据上传提示,还是别处看到的某件事?
> 子提示 b:从你开始用到决定放弃,中间隔了多久?装上就退,还是用了一阵子才退?放弃后换成了什么,还是干脆不用?

**Q3. 为"把代码留在本机 / 自己可控"实际做过的事 + 代价**
为了把代码或工作内容留在本机、自己可控,你过去实际做过哪件事?写最近的一件具体的:自托管了某服务、关掉了某工具的遥测 / 上报、给某 AI 助手设了不上传某些目录、坚持用本地模型而非云端…… 是哪件、大概什么时候。
〔必填:那件事是什么 + 它花了你多少时间或多少钱 + 现在还在维护吗(还是后来放弃了)〕
> 子提示 a:你怎么确认它"真的没把数据传出去"的?去看过日志、抓过包、查过文档,还是凭信任?
> 子提示 b:为这个本地化方案,你具体牺牲了哪些便利或功能?

**Q4.〔本主题核心〕"读我的代码"和"读我跟 AI 的对话",你区别对待过吗**
有没有哪一次,你对工具读取你的**代码**、和读取你**跟 AI 来回聊的对话历史**这两件事,在动作上区别对待过 —— 比如允许它读代码、却不让它碰对话,或反过来?是哪个工具、哪次?
〔必填:举一次具体的 —— 哪个工具、你允许 / 拒绝了哪一类(代码 or 对话)、大概什么时候〕
> 子提示 a:你的对话历史里,最不想被第三方看到的是哪类内容?举一种你实际担心过的(吐槽、试错过程、贴进去过的内部信息……)。
> 子提示 b:有没有哪次你清理过对话记录,或避免在 AI 对话里贴某些东西,就因为怕被别处读到?是哪次?(从没有直接写"没有")

**Q5. 上一次"明知有顾虑、为省事还是交了"**
讲一次你真的破例的经历,不用美化。上一次你明明对"把代码或对话交给某个云端工具"有点顾虑,但为了赶时间 / 图省事还是交了 —— 把一整段含业务逻辑的代码贴进云端 AI、把私有仓库连上某 SaaS、开了全程上云的 AI 录制…… 是哪一次?
〔必填:那次是什么情况 + 当时是什么压力让你顾不上那个顾虑了(deadline?工具太好用?觉得"这段不敏感"?)〕
> 子提示 a:事后你有没有做任何补救 —— 删历史、轮换密钥、把它从某些项目里撤掉?还是就那样了?
> 子提示 b:哪类内容你当时心里划了线、就算赶时间也没贴进去?

**Q6. 一次真实读条款 / 或一次主动脱敏(任选你真做过的那种)**
下面两件高成本的事,挑你过去真做过的那一次讲(都做过则讲最近的一件,并注明哪件):
- (甲)上一次你真的去**读了**某个工具的隐私政策 / 数据条款(哪怕只读关键几句),是为哪个工具?读完你的决定是什么?
- (乙)上一次你把代码或日志交出去前,**先做了脱敏**(删密钥、替换客户名、清 .env、抹内部域名),是为哪个工具 / 哪个场景?你具体抹掉了哪些东西?
〔必填:注明(甲)还是(乙)+ 那个工具 / 场景 + 你最想确认的那句话(甲)或你具体删 / 换掉的东西(乙)〕
> 子提示 a:(甲)你最想从条款里找到的那个答案是什么(是否用于训练 / 是否留存 / 是否本地处理)?(乙)你是手动一条条改的,还是有脚本 / 工具帮你?有没有漏抹、事后才发现的?
> 子提示 b:有没有哪次你想读条款但太长太含糊、或脱敏太麻烦,于是干脆放弃读 / 干脆不交那份东西了?哪次?

**Q7. 一次组织闸门"卡住你",或一次(准)事故改了你的习惯(任选你亲历的那种)**
挑你本人亲历的那一种讲(都遇过则讲影响更大的那次,并注明哪种):
- (甲)你所在团队 / 公司有过一条关于"代码或对话能不能给外部 AI / 云工具"的规定,而你本人被它卡住过一次 —— 当时你是照办、申请例外、还是私下绕开了?
- (乙)你或团队真发生过一次代码 / 密钥 / 会话内容外泄或险些外泄(把 key 贴进云端 AI、私有代码被某服务索引、误发内部信息……)—— 之后你改了什么习惯?

> 总口径见共享方法论。任一题答出"最佳实践 / 一般来说 / 用户应该"即该题 0 分并累计 pundit-flag。

| 维度 | 它回答什么 | 强信号(高分) | 弱信号(0 或低) |
|---|---|---|---|
| D1 阈值真实性 | 隐私边界是不是真在执行的线,而非态度 | 报得出"具体工具 + 时间 + 触发点"并真采取了卸载 / 退订 / 拒授权(Q1 / Q2) | "我一般都注意隐私"却说不出哪一次,或"想过但还在用" |
| D2 已投入成本 | 为本地化 / 可控付出的真实代价 | 自己搭过 server、查过日志 / 抓过包、关过某项遥测,且现在仍在用;为隐私真实支出并说清买的是哪项保障 | "尽量用本地的"举不出做过的事;把功能付费误当隐私付费;"如果有我会考虑付" |

| **D4 对话 ≫ 代码 的敏感差(命门)** | "读 AI 会话"是否比"读代码"更触雷 | 能举出对话里实际担心的内容,且曾因"要读对话"而清理 / 回避 / 停手 | 只停在"感觉会更介意"的假设;说不出对话里担心什么 |
| D5 便利 vs 隐私的临界点 | 什么压力能让他越过边界 + 事后是否补救 | 坦诚说出具体那次破例 + 当时压力 + 事后补救动作 + "划线没交"的内容边界(Q5) | "我从不把敏感东西给云端"(完美人设,可疑)或答成"不建议这么做"的说教 |
| D6 信任跨越证据(采纳开关) | 什么证据能让他从警惕转为采纳 | 说得出让他松手的具体证据(开源可审计 / 明确不留存 / 小范围验证过) | "用着用着就习惯了""大家都用我就用了"无具体证据 |
| D7 组织闸门 | 团队硬约束是否真在执行、能否被绕开 | 说得出规定内容 + 由来 + 本人一次真实被卡 / 申请例外 / 绕开 + 能判断是否真执行 | "我们应该有限制""大公司一般都管"这种不确定 / 二手 |

**立项判定看两组:痛感真实性**(D1 + D5 加权)+ **本主题命门**(D3 + D4,即"读对话"这条线是否真比"读代码"更触雷,决定产品的隐私默认值);**采纳可行性**看 D6 + D7(跨越信任要什么证据、组织闸门挡不挡得住)。每位整理成一张"边界地图"(拒过的 / 为本地化投入的 / 为省事破例的 / 划线绝不交的 / 让他松手的证据),跨人比对找反复出现的同一条线(例:多人独立报出"key / .env 绝不进云端,但业务逻辑片段会贴")。

## 第 5 章 · 竞品切换

> **本章独有研究目标:** 弄清 vibetrace 的**真实替代品**到底是另一款产品,还是"老办法 + 通用 AI"。受访=用过代码上下文 · 记忆 · 历史解释类工具(Pieces、Continue、CodeScene、Cursor memory、Unblocked、repowise 等)的资深开发者,死守"切入→真解决 / 没解决→弃用→回退到什么"切换行为链。

**本章引言(直接发给对方):**
> 嗨,谢谢你抽时间。先把游戏规则说清楚:① 我只要你自己电脑上、自己项目里真发生过的事(哪个项目、哪段代码、大概哪个月)。不要你对这类工具行业的判断,不要"一般来说 / 最佳实践 / 用户应该…",也不要你推荐别人该用什么 —— 你能讲出一套很漂亮的通论,但那对我没用,我要你本人上一次的流水账。② 我不问你觉得哪个工具好、不让你打分、不做功能对比。只问你过去具体怎么切换的。③ 每题〔必填例子〕栏请务必填具体的项目 / 文件 / 时间,空着或写"经常""一般"会让这条作废。④ 想不起来就直说"想不起来"或"没发生过"。全程大概 20–25 分钟。

**本章反陷阱(独有部分):** 本主题最大污染源是资深开发者切换成"评委 / 行业评论员"口吻去点评工具优劣、做功能对比、谈愿景 —— 一旦发现此口吻,该段记二手、降权,第二轮硬拽回"你本人那一次具体怎么做的"。数字栏(第一周打开次数、落空频次、月度金额)若填"经常 / 偶尔 / 看情况"而非数字,该信号作废。

### 资格筛选(1 题,区分第一手 vs 旁观)
**S1.** 过去半年里,你自己电脑上真装过、真打开过哪一个给代码补上下文 / 记忆 / 历史解释的工具(比如 Pieces、Continue、CodeScene、Cursor memory、Unblocked、repowise 之类)?请只写你亲手装并打开过的那一个,别列清单。
〔必填:工具名(单选一个)+ 你大概哪个月第一次用它 + 装在哪个项目 / 哪个代码库上〕
> 子提示:如果你一个都没真装过、只是听说或看同事用过,请直接写"只是旁观,没自己装过"。
> 资格判定:能报出"唯一一个工具 + 具体月份 + 具体项目 / 代码库"=第一手,进主问卷。只能泛指 / 复数 / 说不出时间地点,或自承"旁观"=旁观者,大部分题记二手 / 降权,可只答 Q8(原始工作流)这道不依赖切换经历的题。

### 开放题(8 道)
> 通用提示:每题死锚"你自己上一次 / 那一次"。凡答成"它适合用来…""一般来说…""这类工具通常…"的,都不算数。

**Q1. 切入那一刻 —— 你是怎么开始用它的(频率 / 上次发生)**
回到你"装好它、第一次打开"的那段时间:你是从哪儿知道它的、第一个礼拜实际打开过几次?我想先确认这事真发生过、有多热乎,不是装完就晾着。
〔必填:那个工具名 + 你大概哪个月装的 + 你是从哪知道它的(同事发的链接 / HN / 某条推 / 公司在推)〕
> 子提示 a:装好之后第一个礼拜,你实际打开过它几次?写个数字,别写"经常 / 偶尔"。
> 子提示 b:第一周里你用它干的第一件真事是什么?(写那一件,别写"试了试各种功能"。)

**Q2. 切入扳机 —— 那天你正卡在什么事上,才专门去装它(现有应对 / 已投入)**
回到你"决定开始用它"的那一天:当时你手头正卡在哪件具体事情上,以至于你专门去装了它?描述那天你在做的那个任务 / bug / 那段代码。我不要"它能管理上下文 / 能提升效率"这种功能定位,我要你本人当天遇到的那个麻烦。
〔必填:当天那个具体场景 —— 哪个文件 / 哪个功能 / 什么报错,以及你卡在那儿多久了〕
> 子提示 a:在装它之前,你本来是怎么对付这件事的(翻 git log / 问同事 / 翻 Slack / 直接问 AI)?写下你装它前最后一次用的那个土办法。
> 子提示 b:是什么让你那天觉得土办法不够、非得换工具?写下那个"忍不下去了"的临界点。

**Q3. 它真帮上的那一次 —— value delivered(后果 / 相对优先)**
用了它一段时间后,有没有哪一次它真的帮你把一件事办成了?只写真发生过的那一次,带上是哪段代码 / 哪个问题。
〔必填:那一次 —— 你问了它什么 / 它给了你什么,结果你少做了哪一步,大概哪个项目里〕
> 子提示 a:如果那次没有它,你本来会怎么做、大概多花多久?给个时间估。
> 子提示 b:那之后你有没有把这件事告诉过别人、或在团队里推荐过它?写下你那次实际说了什么、对谁说的(没有就写"没有")。

**Q4. 它没接住的那一次 —— unmet need(现有应对 / 已投入)**
反过来,有没有哪一次你指望它能帮上忙、结果它没接住?那一次你想让它干的具体是什么、它怎么没满足你?只写你亲历的那一次落空。
〔必填:那一次 —— 你想让它回答 / 做的那件事,以及它给出的让你失望的结果,在哪段代码 / 哪个项目上〕
> 子提示 a:那次落空之后,你转头用什么把这件事解决了?写下你那次实际换上的办法。
> 子提示 b:这种落空在你用它的这段时间里发生过几次?给个大概数字,别写"偶尔"。

**Q5. 弃用扳机 —— 你不再开它的那一刻(频率 / 上次发生)**
现在说说"你不再用它"或"用得明显变少"的那一刻:最近一次你打开它是哪个月?之后大概多久就基本不开了?触发你停下来的那件具体事是什么?我要那个让你停手的真实瞬间,不是你事后总结的它的缺点。
〔必填:最后那次打开大概哪个月 + 之后是什么事让你不再开它(卸载了?换电脑没装回?订阅到期没续?某次它惹烦了你?)〕
> 子提示 a:你是一下子停的,还是慢慢就不开了?写下中间发生的那一两件事。
> 子提示 b:停用前后,你有没有改过设置 / 翻过文档想救它一下?写下你那次实际做的挽救动作(没有就写"没有")。

**Q6. 回退到什么 —— true alternative + 真实支出(现有应对 / 已投入)**
停用它之后,这件它本来在帮你做的事,你现在改成靠什么在做?写下你最近一次实际用的那个替代办法(可以是另一个工具,也可以是 git log / 问同事 / 直接问 AI / 干脆不做了)。
〔必填:你最近一次需要"搞清楚某段代码当初为什么这么写"时,具体动手做了什么、用了哪个工具 / 问了谁,大概哪个项目〕
> 子提示 a:这个替代办法,比起当初那个工具,哪点更省事、哪点更费劲?各举你最近遇到的一次。
> 子提示 b:你为这个替代办法现在每月花钱吗(订阅 / API / Copilot 之类)?写下实际金额或"免费";别写愿不愿意付。

**Q7. 切入的隐性成本 —— 授权 / 隐私 / 合规摩擦(后果 / 相对优先)**
有没有发生过这样一次:你为了用上某个这类工具,得让它读你的代码库 / 接进公司仓库 / 登录账号,而你当时犹豫了、或被拦下了?那一次具体是什么情况?只写你本人或你团队真碰到的那一次。
〔必填:那一次 —— 哪个工具要什么权限 / 要把什么数据传出去,谁(你自己 / 安全团队 / 老板)在哪一步拦了或犹豫了〕
> 子提示 a:最后那次是装了、放弃了、还是找了变通(本地版 / 脱敏 / 只在私人项目用)?写下你实际的处理。
> 子提示 b:你装它之前有没有去看过它把代码传到哪、存多久?写下你那次实际查了还是没查。

**Q8. 原始工作流 —— 你最近一次还原代码决策实际跑了哪些步(现有应对 / 已投入)**
(此题不依赖你是否用过上述工具,人人可答)最近一次你接手别人(或半年前的自己)写的、看不懂为什么这么写的代码,是哪一次?那次你为了搞清楚"当初为什么这么决定",实际跑了哪些步骤?我要那一次你真做的动作流程。
〔必填:那段代码大概干什么的 + 你按顺序做了哪几步(翻 blame?翻 PR 讨论?翻 Slack / 工单?直接问写的人?问 AI?),大概哪个项目 / 哪个月〕
> 子提示 a:这几步里最耗时 / 最让你卡的是哪一步?写下那一步当时的具体情形。
> 子提示 b:最后你搞清楚了吗?搞清了 / 猜了个大概 / 还是放弃照着改了?写下那次的真实结局。

### 第二轮深挖(按首轮答案择题追问)
> 第二轮只挑首轮答出了具体事件的题往深里挖一层;首轮泛化 / 跳过的不追。每次只问 2–3 个。

- 承 Q1 / Q2(切入):那个"忍不下去了"的临界点之前,你其实已经忍了多久?这件事卡你之前,在更早的项目上也卡过吗 —— 还是这次特别严重?(挖痛的频次与累积,判断是真痛点还是一次性新鲜感。)
- 承 Q3(真帮上):那次它帮你省下的那一步,现在没有它了,你是又退回去手动做、还是找到别的替代?(验证"被帮到"是不是真不可替代。)
- 承 Q4(没接住):它没接住的那件事,和 Q3 它帮上的那件事,是同一类需求吗?如果只能让它做好一件,你会要它先把哪件做对?用这两次的事说明,别讲一般道理。(逼出同一品类内的相对优先级。)
- 承 Q5 / Q6(弃用→回退):你回退到的那个办法(git log / 问人 / 问 AI),其实在你装那个工具之前就一直在用吧?那当初那个工具到底比这个老办法多给了你什么、又少了什么 —— 各举一次。(揭示真正的对手往往是"老办法 + AI"。)
- 承 Q6 支出:你报的那个月度金额,是你个人掏的还是公司报销?如果哪天要砍预算,这笔和你其它开发工具订阅比,会先砍哪个?(用真实排序而非意愿,测真实优先级。)
- 承 Q7(隐私摩擦):那次被拦 / 犹豫之后,有没有因此就彻底没用上某个你本来想用的工具?那件它本来要帮你做的事,后来怎么了?(把隐私摩擦量化成一次真实流失。)
- 承 Q8(原始工作流):你说的最卡那一步,最近三个月里又遇到过几次?最近的那次是什么时候、哪段代码?(把核心 JTBD 的发生频率落到可数。)
- 补问(若首轮提到"装了没盘活"或"团队推了我没跟"):把那一次单独讲清 —— 卡在哪一步(要登录?要喂代码库?要配置?第一次结果不对?),卡住那刻你是去翻文档还是直接关掉?(挖 onboarding 摩擦与 time-to-value。)

### 本章判定 / 优先级维度
> 总口径见共享方法论。一题里若混了具体事件 + 通论,只就事件部分计分。

| 维度 | 看什么(只取过去行为) | 强信号(高分) | 弱信号 / 记 0 |
|---|---|---|---|
| 切入扳机真实性 | Q1 / Q2:具体当天痛点 + 装前旧办法 + "忍不下去"临界点 | 当天任务可名状 + 旧办法 + 明确临界点 | "为了提效 / 想试新东西 / 它能管理上下文" |
| 真交付价值 | Q3:一次具体成功 + 省了哪一步 / 多少时间 + 是否主动安利 | 单次成功 + 量化省时 + 安利过 | "整体挺好用 / 理论上能帮回溯" |
| 未满足需求 | Q4:一次具体落空 + 当场替代办法 + 落空频次数字 | 具体落空 + 即时替代 + 频次是数字 | "大库可能不准 / 据说某语言不好" |
| 弃用扳机 | Q5:最后使用月份 + 具体导火索 + 是否尝试挽救 | 月份 + 导火索 + 挽救动作 | "用着用着忘了 / 没什么原因 / 用处不大了" |
| 真实替代与支出 | Q6:最近一次回退的实物动作 + 真实月度金额 | 可名状回退动作 + 真实数字(含"免费") | "看情况 / 有很多种方式";愿不愿付=0 |
| 切入隐性成本 | Q7:一次真实授权 / 合规摩擦 + 具体拦截方 + 实际处理 | 真实摩擦 + 拦截方 + 处理 + 是否查过数据去向 | "数据安全当然重要 / 企业都在意隐私" |
| 核心 JTBD 强度 | Q8:一次具体陌生代码场景 + 有序真实步骤 + 最卡那步 + 真实结局 | 具体场景 + 流水账步骤 + 卡点 + 结局 | "一般先看 blame 再… / 应该写好文档" |

**关键判读(超越逐题分):** ① **真正的竞争对手** —— 统计 Q6 回退去向;若多数人回退到"git log + 问人 + 直接问 AI"而非另一款产品,则 vibetrace 的对手是"老办法 + 通用 AI",定位与上手成本须据此校准。② **切入 vs 留存漏斗** —— 把 Q1(第一周打开次数)、Q5(弃用导火索)、补问(装了没盘活)拼成一条漏斗,定位流失集中在 onboarding 摩擦、time-to-value、还是价值不可持续。③ **隐私切入点验证** —— Q7 出现真实摩擦事件的比例 = vibetrace"本地优先、数据不出本机"卖点的需求印证强度(只数真事件,不数口头表态)。

## 第 6 章 · 功能优先级

> **本章独有研究目标:** 在 vibetrace 的八个底层任务 **A–H** 里,定出最该**先打磨谁**。功能优先级 = 任务优先级(一个任务可对应多功能)。映射:A→ask / blame、B→graph、C→watch / 时间胶囊、D→self、E→brief / console、F→tunnel、G→digest、H→course。**全程不暴露任何功能名**,只让真实任务在受访者自己的烦恼清单里自发浮现。

**本章反陷阱(独有部分):**
- **绝不给功能清单让排序。** 开场纯空白让对方自发捞;"排第一"只排他自己列的真实烦恼,不是排你的功能清单;收尾"最想解决的"排序对象同样是他列的真实任务,不是 vibetrace 功能。
- **付费问句只问过去真实支出**("你付过吗 / 订过吗"),绝不问"你会不会为能自动生成日报的工具付费"—— 后者是意愿 + 假设,信号全废;若答"没付过但我觉得值得付",记 0。
- **高频 ≠ 高痛。** E(开工进上下文)类天然高频,必须靠真实拖延 / 漏做后果判断,别被"每天都会重启"的高频假象误导,severity 维度据此降权。
- **C(回去验证当初担心的风险)是最易"听起来好但没人做"的任务** —— 严禁暗示"你应该回去验证";问句必须两步走:先确认担心(过去)、再问有没有真回去看(过去),允许对方诚实回答"从没回去看"。

### 资格筛选
沿用通用资格口径(过去 3 个月亲手写 / 改过、需自己回看的活代码;旁观 / 管理者降权)。本章可直接用第 1 题作为暖场兼筛选 —— 列不出任何真实开发烦恼者本卷价值有限。

### 开放题(7 道)
> 排序逻辑:开场用纯空白自发诊断题(易答 + 定真实基调)→ 中段按"开场最可能自发冒出"顺序逐个任务锚定深挖(A 核心主张先行,E 高频日常,B / C 较稀疏 / 特殊主张取证频率)→ 合并 D + G 处理一段时间产出的复盘与成品(顺带 F / H 演化回看 + 唯一的过去付费探针)→ 收尾用已投入交叉验证题回收开场闭环、决出相对优先级冠军。

**Q1 ·〔自发进榜诊断,核心开场〕真实烦恼清单**（对应全部 A–H）
先不聊任何工具。回想上周或最近几天,真正吃掉你时间、让你烦躁或卡住的开发瞬间有哪些?随手列 2–4 个,每个一句话说清当时在干嘛、卡在哪。如果按"最想这辈子别再遇到"排,哪个第一?哪个是反复出现的老问题?
> 子提示:有没有哪个当时让你直接停下、改去干别的?是哪个、当时绕开它去干嘛了?哪个是头一回、哪个是老毛病又犯了?
>
> 〔A / B / C / E 等底层任务是否自发进榜并排前;能否报出具体场景而非泛泛抱怨;哪个被点名"最想别再遇到"。若列表全是环境配置 / 依赖冲突 / 纯算法难题等与回溯历史无关的事,说明这些任务在真实痛感里不靠前。绝不在此暴露任何功能或任务方向。〕

**Q2 · 想不起当初为什么这么写**（对应 A:ask / blame)
想最近一次:你或 AI 写过的一段代码,过一阵回头看搞不清当初为什么这么写、当时基于什么定的。最近一次是哪个项目、哪段、大概什么时候?当时你做了什么去搞清楚,最后弄明白了没有?
> 子提示:你具体做了哪些动作找答案(翻 git log / 旧对话 / 重读 / 问人 / 贴回给 AI 重解释)?花了多久?这种情况多发生在你自己写的老代码,还是 AI 当时替你写 / 替你决定的那部分?如果是贴回给 AI 重解释:你信几分?后来发现它编 / 搞错了吗?那个错代价多大?近一两个月还有几次类似的?分别哪些项目?
>
> 〔能立刻报出具体项目 + 代码 + 近期时间且能再枚举 2 次以上;真动手翻 log / 旧对话且花可观时间;不确定造成过返工或改坏;尤其指向"AI 当时替我做的决定我现在看不懂"。低信号:含糊"偶尔吧"、举不出第二例、"重写更快从不去查"。〕

**Q3 · 重新打开搁置项目、捡回上下文**（对应 E:brief / console)
想最近一次你重新打开一个搁置了几天 / 更久的项目准备继续干。坐下那一刻,为了想起上次停在哪、接下来先碰哪块,你实际做了哪些事?哪个项目、隔了多久没碰、花了多久才进入状态?
> 子提示:具体翻了什么找回状态(git log / 上次 AI 对话 / 没提交的改动 / TODO / 笔记 / 全靠脑子)?哪样最不顶用?收工前你会不会特意给明天的自己留点线索?这习惯坚持下来了吗?有没有哪次因为捡不起上下文,干脆当天没动那个项目,或漏了 / 重复做了上次的一块?过去一两周这种重进状态发生过几次?
>
> 〔几乎每次重启都花时间找回状态、近期高频、已自发形成收工留线索的习惯(痛到自救)、发生过漏掉 / 重复 / 当天弃坑的实际损失。低信号:坐下就接着写、靠脑子够、就一个项目天天在上面不存在断片(高频但无痛,优先级反降)。高频 ≠ 高痛。〕

**Q4 · 一次改动牵连出一连串别处也得改**（对应 B:graph)
想最近一次你改了某处、发现它牵连出一连串别处也得跟着改(像涟漪扩散)。最近一次哪个项目、改的什么?当时你怎么搞清楚这改动还牵动了哪些地方的?有没有漏掉某处直到后来才暴雷?
> 子提示:怎么把受牵连范围找全的(全局搜 / 跑测试看哪挂 / 凭经验 / 问 AI / 一点点踩出来)?牵连是改的当下意识到,还是事后(跑测试 / 上线 / 别人指出)才发现?隔了多久?事后(几周后)你有没有再回去看过它总共牵动了多少?还是改完就过去了从不回看?过去几个月这种连锁改动发生过几次?哪类改动会引发(重构 / 换库 / 改数据结构 / 改接口)?
>
> 〔近期有具体连锁事故 + 找范围吃力或漏了导致暴雷 + 事后确实想看"总共牵动了啥"。低信号:项目小一眼看清、有测试兜底坏了再修、find usages 就够、想不出具体一次(任务 B 稀疏→graph 优先级下调)。graph 是产品里较可能稀疏的任务,本题唯一目的是取证它真实频率,别引导对方承认需要影响图。〕

**Q5 · 做技术决定时打鼓的那个隐患后来成真没**（对应 C:watch / 时间胶囊)
想最近一次你做某个技术决定时心里打鼓 —— 担心"这么搞以后可能会出问题"(临时方案 / 赌的假设 / 性能 / 并发)。最近一次担心的是什么、哪个项目?后来你有没有真回去看过当初担心的那件事到底成真了没有?
> 子提示:是你主动回去查了、还是等它真出问题才发现、还是到现在也没回去看?为了不忘记回去验证,你当时有没有专门记在哪(代码 TODO / 笔记 / 提醒)?管用吗,还是照样漏掉?把范围放宽:过去几个月有几次是你当初担心的隐患后来真的爆了?爆的时候你还记得自己早担心过吗?
>
> 〔能举出具体一次"担心→后来真回去验证或被反咬"完整事件,且承认"常忘了自己担心过、等爆了才想起"(忘了 + 真出事=痛点真实且无人接管,高优先)。低信号:"担心的我都当场修掉不留""从不回头出事再说"、或根本举不出任何一次成真。这是最易"听起来好但没人做"的任务,严禁暗示"你应该回去验证";两步走,允许诚实回答"从没回去看"。〕

**Q6 · 跟人 / 跟自己交代"这段时间到底干了啥"**（对应 D + G:self + digest)
回想最近一次你需要跟人(或跟自己)交代"过去这段时间这个项目 / 我和 AI 到底干了啥、产出了啥"(写周报 / 跟协作者同步 / 自己复盘 / 交接)。那次你怎么把这段时间发生的事重新拼出来的?
> 子提示:从哪几处东拼西凑(git log / 翻 AI 对话 / 翻 PR / 凭记忆 / 已有的流水账习惯)?最难拼的是回忆改了啥,还是回忆为什么这么改?最后那份东西给谁看了(同事 / 客户 / 老板 / 未来的自己 / 没人看走流程)?整理花了多久、最烦哪一步?为这类整理 / 回看的事,你现在或过去有没有实际付过钱(订过工具 / 服务)、大概多少?有没有为了交接 / 面试 / 文档,沿时间线把整个项目演化从头回看过一遍?那次怎么还原整段历史、花了多久?
>
> 〔确实定期需要交代 / 复盘 + 东拼西凑很费劲且漏过事 + 尤其卡在"为什么这么改"而非"改了啥";若已为相关工具付过费=最强沉没成本信号;项目演化回看有具体场景且硬扛过整段历史(覆盖 F / H)。低信号:单干没人要汇报、从不写周报、git log 扫一眼就够、从没付过钱、从没回看演化。付费问句只问过去真实支出,绝不问"你会不会为能自动生成日报的工具付费"。〕

**Q7 ·〔已投入交叉验证,核心收尾〕折腾最多仍没解决的那个**（对应全部 A–H)
回到你第一题列的那几个烦恼。这些事卡住时你现在事实上怎么对付的?哪个是你已为它折腾过最多次、试过最多种办法(查资料 / 记笔记 / 写脚本 / 问 AI / 改流程)却还是没真正解决的?讲讲都试过啥。
> 子提示:试过的办法里哪个用了一阵又放弃?为什么放弃?若这事这个月一次都没发生,你会明显轻松吗?还是其实也还好?和第一题列的其他烦恼比,它真是最想解决的吗?若不是,更想解决的那个你又为它折腾过什么?
>
> 〔能指出某个反复折腾、试过多种办法仍没解决的任务,且去掉它这个月会明显更轻松(高投入 + 未解决 + 去掉真减负三者齐全=最该先打磨)。注意它对应哪个底层任务(A–H)=相对优先级冠军。低信号:其实没怎么专门折腾、忍忍就过、去掉也没差。在他自己的清单里指认,而非给功能选择。〕

### 第二轮深挖
> followup 按对方答案择一两条追,不要清单式逼问。第 6 题付费分叉若答"没付过但我觉得值得付"→ 记 0(意愿不是行为)。对方主动夸功能 / 提需求,不接、不确认,拉回"上次具体发生那次你做了啥"。

各题的深问已内嵌于上方子提示;第二轮重点把 Q1 自发进榜的前两个任务,与 Q7 指认的"投入最多仍没解决"任务对齐,锁定相对优先级冠军。

### 本章判定 / 优先级维度(任务 A–H 三维评分 · 乘法)
> 访谈后对每个底层任务 A–H 打分,功能优先级 = 任务优先级。三维全部只从过去行为取证,口头偏好 / 付费意愿一律记 0。

| 维度 | 由什么反推 | 3 | 2 | 1 | 0(直接出局 / 封顶) |
|---|---|---|---|---|---|
| **频率 freq** | "近一两个月 / 周枚举次数" | 每周多次且枚举具体多例 | 每月数次 | 举得出 1 次但稀疏 | 举不出任何具体一次→该任务直接出局 |
| **后果严重度 severity** | "上次卡住后真实发生了什么" | bug / 线上事故 / 大范围返工 / 弃掉一条线 | 吃掉可观时间或反复绕 | 轻微烦但无实损 | 当场化解无后果 |
| **已投入 sunk(最强信号)** | "为它折腾过什么" | 已付过钱 / 订过工具,或自建机制且仍没解决 | 反复试过多种凑合办法 | 临时手动凑合 | 从不专门折腾、忍忍就过 |

**任务得分 = freq × severity × sunk(乘法)** —— 任一维为 0 即接近出局,体现"高频但无痛""听起来好但从没做""没人投入"都不该先打磨。

**归因校验:**
- **自发性加权:** 开场第 1 题自发进榜且被点名"最想别再遇到"的任务 ×1.3;只在追问后才出现的任务 ×0.8。
- **决胜局:** 收尾第 7 题指认的"投入最多仍没解决 + 去掉这月会明显轻松"的任务,即相对优先级冠军。
- **反夸大:** 若"去掉它这个月你会明显轻松吗"答"其实也还好",severity 封顶 1。
- **任务→功能落地:** 得分最高的任务,选其当前凑合手段最重(动作链最长 / 最费时)的那个功能先打磨;一个任务有多功能时,优先打磨能直接顶替对方现有凑合动作的那个。
- **覆盖提醒:** F(tunnel)/ H(course)本轮仅靠合并 D + G 题的演化回看 followup 取证,样本天然偏少,除非多人自发强提,默认排在 A / E / D-G 之后,标记为"证据不足待补访"而非"低优先"。
