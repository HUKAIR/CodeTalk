## 每阶段要点
- **0 二手坐实**:①②③ 直接引公开调查(见证据底座),**不再花访谈证痛点**;数字外引前回原始来源核对。
- **招募**:1–3 名重度 AI 编码者本人即一线写码 / 决策;**高 + 低后果都要有**(同样「信任位移」商业含义不同,别混算)。
- **真实性门槛**:一条经历须报得出 **具体项目 + 文件 / PR + 大致时间**;含糊 → 中性追问「最近**具体**哪一次?」**别替他举例**。
- **反 pundit**:答「最佳实践 / 一般来说 / 用户应该」且报不出具体一次 → 该题记 0;累计 ≥3 次整份样本降级。
- **盲法(B 段)**:甲 / 乙**随机换序**、别说哪份是我们的;先 B1(哪句像编)再 B2 打分(分数才有锚)。key:甲=AI 反推、乙=真实记录(`4440324`,长度已配平)。
- **C 段(最硬,尽量拉到)**:作者即真相,唯一能证伪「反推会编 / 会漏」。脱敏话术:「**可只发结构性 diff / 打码业务字段,我们只看为什么这么改、不看业务内容**」。
- **证据纪律**:单人 ≠ 人群,升人群门槛 = 同结论 **≥3 人复现**;N<3 **绝不**写「用户们 / 普遍」。文案落点挂「找回 why / 防踩坑」,不挂「记录 / 留痕 / 省时间」。

## 纠偏 & 边界(必记)
- **🚩 别把 survey 的「普遍不信 AI」当「我的差值已验证」**:前者是市场背景(二手坐实痛点),后者是产品有效性(只能真人验差值)——把背景当验证 = 没地基盖楼(pm-brief 第 14 行已钉死)。
- **痛点 ①②③ 二手坐实即可、真人只验 ④⑤**:别再花访谈 / quota 去证「开发者普遍不信 AI / 丢上下文 / AI 会编」。
- **「6 分」是问卷1 单人 N=1**(问卷2/3 查无打分)→ 验差值方向 ≥3 人复现,不预设 6、不说「破天花板」。
- **D 实付=0 不否决 G2**(信任位移=G2;付费=G3;脱钩,不挂北极星 / 付费墙)。
- **隐私是减分项不是付费理由**,别测付费意愿;freelancer 的 C 段必给脱敏话术,否则最高价值样本流失。
- 对齐 `ROADMAP.md:40` 时序闸:G1 拦截(`interceptions.md` 现 0)/ G2(本 SOP)/ G3 飞轮,三门同绿前 NEVER 启付费墙。
