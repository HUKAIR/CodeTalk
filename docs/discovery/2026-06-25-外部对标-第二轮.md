## 一句话:第一轮证明「市场空白存在」,第二轮给了「为什么是你」的硬弹药 + 一记定位警告

- **硬弹药**:首个独立学术实证(MSR 2026)证伪了厂商「AI 审查能独立处理 80% PR」,AI-only 审查合并率只有 45.20%、六成 PR 信噪比落在 0–30%;同时 RAG「接地」有了可验收的学术定义,而你的**零-LLM 逐字引用能以字符串恒等确定性满足它**——这是生成式 LLM 证明做不到的。
- **定位警告**:「接地真实数据对抗幻觉」这句口号**已被 GitKraken/Context7/GitHub MCP 占用**,不再是差异化。差异化必须收窄到「**零-LLM 确定性 + 真实 commit/PR/会话原话逐字引用 + 自动挖掘(非手写)+ 数据不出本机**」这个复合体。

## 焦点 1 · code-review 理解力(新角度)

### 研究锚点
- **首个第三方实证证伪厂商宣传** `[数据]`:MSR 2026(arXiv 2604.03196,同行评审录用)基于 AIDev 19,450 PR:AI-only 审查 PR 合并率 **45.20% vs 人工 68.37%(−23.17pp,p<0.001)**,弃用率 34.88% vs 21.60%,**60.2% 的 AI-only PR 信噪比在 0–30% 区间**,12/13 审查 agent 平均信噪比 <60%。⚠️ 论文自述**关联非因果**(AI-only PR 复杂度可能不同),引用须说「关联」。
- **「commit 即决策记录」结构化新协议** `[机制]`:**Lore**(arXiv 2603.15566,2026-03,有 GitHub/npm 实现)用原生 git trailers 把 commit 重构成自包含决策记录(约束 / **被否决的备选** / **agent 指令** / **验证元数据**);**git-adr**(Rust v1.0)把 ADR 存进 git notes + `git adr link <adr-id> <commit>` 做 commit 级绑定。
- **ADR 生态碎片化 + 全手写** `[数据]`:官方目录(adr.github.io)并存 Nygard/MADR/Y-Statement/42010 多套模板,主流工具(adr-tools/Log4brains/pyadr…)全是 CLI+markdown,**无一从 git 历史自动挖掘 why**。
- **MCP 生态已有钦定基座 + 本地竞品** `[机制]`:**GitHub 官方 MCP server**(`get_commit/get_diff/get_commits/search_pull_requests`,但出网 PAT→API);**codebase-memory-mcp**(14.2k stars,本地 SQLite 知识图,但 ADR 手填、不锚定真实 commit/PR/原话、不追 why-provenance)。

### 三视角
- **技术专家**:① **MCP 工具签名向 GitHub MCP 对齐**(`get_diff/get_commits` 已是事实标准,agent 复用零学习成本),但守本地红线(你不出网是差异)。② **借 Lore 扩面包屑 schema**:你的 `Vibe-Decision:`/`Vibe-Watch:` 与 Lore 高度同构,可补「被否决的备选 / agent 指令 / 验证元数据」字段,参考 git-adr 的 commit-link 原语做行级锚定。
- **AI 开发者**:MSR 2026 是「AI 反推式审查噪声大」的独立证据——把「确定性引真实原话」定位成**对噪声/幻觉的解药**(说「关联」)。codebase-memory-mcp 证明本地 repo-memory 赛道已有 14.2k-star 玩家,但它拼的是结构化代码图谱 + 手填 ADR,**why-provenance 仍空**——别去拼代码图谱,聚焦真实原话溯源。
- **产品经理**:**最干净的一句差异化 = 「别人要你手写 why,我自动从真实 commit/PR/会话原话挖出来并接地」**。ADR 生态把 why-capture 当一次性手工负担(高摩擦、易过时)→ 你的反向「零手写、自动 provenance」正填此空位。可加 **MADR/Nygard 导出**降团队迁移摩擦;碎片化意味着不必押单一 schema。

## 焦点 2 · 前端 UX(新角度)—— 直接对齐 GitLens,别发明交互

### 研究锚点 `[机制]`
- **GitLens 已确立开发者专属行级溯源范式**(非客服 chatbot):gutter/行尾 **unobtrusive blame 注解**(谁/何时/哪个 commit 改了此行)+ **CodeLens**(文件顶/代码块头显示 Recent Change + Authors)+ **hover card 渐进式披露**(commit 富细节 + 可点 SHA 跳 Show Commit、PR/issue 链接)。来源:GitLens README + GitKraken 文档 + VS Code CodeLens API。
- 通用 AI 信任 UI 模式(shapeof.ai / aiuxdesign.guide):可点击内联引用、hover 预览来源、置信度可视化。

### 三视角
- **技术专家 / AI 开发者**:行级 why 应走 **gutter 注解 + CodeLens「为什么这么写」入口**,而非弹聊天框——这正是第一轮 NN/g「单一入口 + 渐进式披露」的开发者工具落地形态。引用核验直接复用 **hover card**:hover 预览真实原话、点引用跳真实 commit/PR/会话原文(= 第一轮 SO/Uber「引用来源 + 暴露置信度」的 dev-tool 形态)。**你已有的可点开核验引用 chip 是对的,下一步是 hover 预览 + 跳源。**
- **产品经理**:**最高保真形态是 IDE 扩展**(VS Code,贴 GitLens 肌肉记忆:gutter「为什么」+ hover 真实原话 + 点击核验),比独立 web 控制台更贴 review/编码现场。这是与你「MCP 底座」并行的第二个分发面候选——但要权衡 M0 红线与维护成本(留作评估,非现在做)。当前 web console 内嵌 chat(本轮 PR #52 已做)是务实第一步。

## 焦点 3 · 接地质量 + 变现(新角度)

### 研究锚点
- **接地质量有可验收的学术定义** `[数据]`:arXiv 2409.11242(Trust-Score)——一个 RAG 响应「接地」当且仅当**仅用检索文档正确作答 且 每条陈述可由其内联引用推得**。**你的零-LLM 逐字引用真实原话能用「字符串恒等」确定性满足「可由引用推得」判据**,绕开生成式 LLM「零幻觉不可达」的不可能性结论。⚠️ groundedness ≠ 事实正确(源本身可能错):你接地的是「当初到底说了什么/为什么这么写」,不是客观真理——这是正确 scope,**别宣传成「保证代码正确」**。
- **「接地对抗幻觉」口号已被占用** `[机制]`:GitKraken MCP 落地页明牌「grounding in accurate Git history instead of hallucinations」,Context7 / GitHub MCP 同款话术。

### 三视角
- **AI 开发者**:把「零幻觉」做成**可量化验收门**——答案须被引用的真实原话蕴含(字符串恒等),而非 LLM-as-judge 噪声。可接 RAGAS/DeepEval 的 groundedness 指标做接地对话回归(待第三轮定具体框架)。这给护城河一个**可写进文案的技术正当性论证**。
- **产品经理(定位校准 · 重要)**:「grounding vs hallucination」不再是差异化叙事(多家占用)。**收窄到复合护城河**:零-LLM 确定性 + 真实 commit/PR/会话原话**逐字**引用 + **自动挖掘**(非手写)+ 数据不出本机。其中「逐字 + 自动 + 满足 Trust-Score 判据」是别人给不出的组合。
- **变现**:⚠️ **本轮最薄**——只拿到定位/叙事信号(codebase-memory-mcp 走 MIT 开源 14.2k stars),**定价/按席位 vs 用量/团队版触发点全无硬证据**。留作第三轮专题。
