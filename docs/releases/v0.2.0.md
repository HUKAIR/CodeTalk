# CodeTalk 0.2.0

CodeTalk 0.2.0 makes local pre-commit decision review the primary workflow and
makes every optional remote enrichment request inspectable before authorization.

## Primary Workflow

CodeTalk now reviews a local diff against original decision records before the
change ships. A review card keeps authored evidence, generated interpretation,
provenance precision, and maintainer judgment separate. The local browser
supports four outcomes; only a confirmed conflict that changes the proposed
action is a verified interception.

Optional enrichment is plan-first. A configured key does not authorize remote
transfer. The plan shows the exact destination, bounded inputs, redaction
counts, data still visible after redaction, cache effects, and the provider
retention boundary. Remote execution requires `--allow-remote` for that command.

## Integrity

`SHA256SUMS` covers every distributable and the CycloneDX 1.6 SBOM. The SBOM
records the same SHA-256 digest for each Python, MCP, and editor artifact. The
GitHub Release is published only after all assets are attached to its draft;
release immutability then locks the tag and assets and produces a verifiable
release attestation.

## Known Limitations

- **cold-start evidence gaps:** repositories without decision notes or optional
  enrichment initially expose little more than commit subjects.
- **human semantic judgment:** deterministic associations can surface relevant
  history but cannot decide whether a current change truly repeats it.
- **provider retention:** remote model retention is controlled by the selected
  provider and remains outside CodeTalk guarantees.
- **unofficial local session formats:** Claude Code, Cursor, and Codex session
  formats can change without notice; parsers degrade to the evidence available.
